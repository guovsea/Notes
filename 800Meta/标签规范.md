# 标签规范

## 核心原则

- **文件夹解决"放在哪"，标签解决"是什么状态/属于什么上下文"**
- 不要用标签重复文件夹结构（已在 100Cpp 就不加 `#cpp`）
- 不要用标签做层级分类（`#编程/前端/React` 是文件夹的工作）
- 标签的价值：打破文件夹的物理边界，从不同维度找到同一批笔记

---

## 1. 状态标签 — 笔记生命周期

| 标签 | 含义 | 适用场景 |
|------|------|---------|
| `#todo` | 待完善 | 内容不完整、需要补充 |
| `#draft` | 草稿中 | 正在写、还没定稿 |
| `#done` | 已完成 | 内容完整，参考用 |

## 2. 类型标签 — 文档体裁（跨文件夹）

| 标签 | 含义 | 适用场景 |
|------|------|---------|
| `#howto` | 操作指南 | 工具配置、环境搭建、CMake 用法 |
| `#snippet` | 代码片段 | 可复用的代码模板、算法模板 |
| `#research` | 调研报告 | 对某个主题的系统性调研 |
| `#reference` | 速查/参考 | 语法速查、API 参考、命令清单 |
| `#diary` | 日记 | 带日期的个人记录 |
| `#idea` | 灵感 | 还没展开的想法、快速记录 |

## 3. 上下文标签 — 横向关联（打破文件夹边界）

| 标签              | 含义   | 连接的内容                                     |
| --------------- | ---- | ----------------------------------------- |
| `#interview`    | 面试相关 | 面试准备、算法题、面试题                              |
| `#leetcode`     | 刷题   | LeetCode 题解、算法练习                          |
| `#pitfall`      | 踩坑   | 环境问题、兼容性、Bug 记录                           |
| `#reading-code` | 读源码  | 开源项目阅读笔记                                  |
| `#fem`          | 有限元  | 数学（梯度/散度/拉普拉斯）、物理（弱形式）、项目（poisson solver） |
| `#series`       | 系列笔记 | 有编号或顺序的多篇关联笔记                             |
| `#ai-assisted`  | AI 协作 | 内容不完全由自己撰写，经过 AI 辅助生成或改写                  |
