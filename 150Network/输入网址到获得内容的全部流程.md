## 基础检查
1. 浏览器检查网址的正确性
2. 检查本地是否存在缓存，如果存在缓存则直接加载缓存

## DNS 解析获取 IP 地址
3. 检查浏览器 DNS 缓存中是否存在 DNS 信息，检查主机的 `hosts` 文件，如果存在缓存则能直接获取其 IP 地址
4. 向本地 DNS 服务器发送 DNS 请求，本地 DNS 也存在缓存策略
5. 如果本地 DNS 服务器缓存中不存在的话，则以迭代的方式分别向：DNS 根服务器、DNS 顶级域服务器、DNS 权威服务器获取下一层级的 IP 地址，最终查询到网址真实的 IP 地址

## TCP 连接
6. 在发送 HTTP 请求前，需要先通过 [[TCP三次握手]] 建立与服务器的 TCP 连接（HTTPS 还需进行 TLS 握手）

## 封装
7. 逐层封装 HTTP 报文：HTTP 协议使用 TCP 协议，HTTP 报文在运输层会被加上 TCP 头部，然后网络层再加上 IP 头部，最后在数据链路层被加上帧头和帧尾

## 发送
> 发送过程可分为局域网内和局域网间

8. 局域网内：发送端查看目的 IP 地址是否存在于 MAC 转发表（ARP 缓存）中，如果存在则在帧头部添加目标 MAC 地址并从相应的端口发出
> 如果目标 IP 地址不在转发表中，则使用 ARP 协议广播该 IP 地址，包含该 IP 地址的接收端做出回应，回应它的 MAC 地址
9. 局域网间：需要通过路由器进行发送，路由器中有一个路由表，记录着每个 IP 地址（网段）所对应的端口，当接收到某个 packet 时，会根据其目的 IP 地址选择某个端口进行转发

## 解封装与响应
10. 服务器端在接收到数据后会逐层解封装，最终将 HTTP 报文发送给上层应用，上层应用根据 HTTP 报文做出响应，将客户端请求的网页发送到客户端，然后通过浏览器对网页内容进行渲染
