## 相关

- [[符号和符号表]] -- 强符号与弱符号的定义

> 编译器不允许一个编译单元中存在多个同名符号
> 但连接器允许多个模块（可重定位目标文件，obj,dll,libso）同时存在同名符号

## 强符号和弱符号

函数和已初始化的全局变量是强符号，未初始化的全局变量是弱符号。编译器会将这个信息隐含在符号表中


```cpp
int i = 1;
int j;
int k = 0;
```

```shell
➜  objdump -t libmine.so

libmine.so:     file format Mach-O 64-bit x86-64

SYMBOL TABLE:
0000000000004000 g     O __DATA,__data _i
0000000000004004 g     O __DATA,__common _j
0000000000004008 g     O __DATA,__common _k
0000000000000000         *UND* dyld_stub_binder
```

- COMMON 表示还未分配位置的符号：未初始化的全局变量。编译器无法确定链接器会使用多重定义的哪个符号，将决定权留给链接器
- `.bss` 静态变量，静态变量全局唯一，因此不允许有多个同名符号


## 规则

- 不允许有多个同名强符号
- 如果有一个强符号和多个弱符号同名，让所有弱符号引用这个强符号
- 如果有多个弱符号同名，那么任意选一个弱符号

由于规则 2 和规则 3，一个模块可能会不经意间修改另一个模块中的变量

```cpp
# foo.c
int x = 123;
```

```cpp
# bar.c
int x;
void f() {
    x = 1;
}
```

可能想修改本模块中的变量，结果同时也修改了其他模块中的变量

更可怕的错误：

```cpp
# foo.c
int x = 123;
int y = 456;
```

```cpp
# bar.c
double x;
void f() {
    x = 3.14;
}
```

这会让 foo 模块中的 x y 变成非常奇怪的值，因为这会让 x 和 y 的内存被一个双精度浮点数覆盖。

幸好，链接器可以检测出多重定义符号的问题并发出警告，可以使用 `-fno-common` 视为错误
