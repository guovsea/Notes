#### 符号和符号表

局部非静态变量没有符号，因此也可以说它具有 [[链接属性 Linkage#No linkage | No linkage]] 的链接属性。

静态局部变量存在符号，并且在 .data 或 .bss 段中。编译器通过在符号名中编码函数名来区分不同函数中的同名静态局部变量（如 `_f.a`、`_g.a`）。


## objdump 查看符号表

```shell
objdump -t <目标文件>
```

## 源文件

```cpp
int a = 0;
int b = 1;
int f() {
    static int a = 1;
    static int b = 0;
    return b;
}
int g() {
    static int a = 1;
    return 0;
}
```

编译为动态库

```shell
gcc mine.cpp -o libmine.so -shared -fPIC
```

## 符号表

```shell
objdump -t libmine.so
```

```shell
➜  staticLocalVar objdump -t libmine.so

libmine.so:     file format Mach-O 64-bit x86-64

SYMBOL TABLE:
0000000000004004 l     O __DATA,__data _f.a
0000000000004008 l     O __DATA,__data _g.a
0000000000004010 l     O __DATA,__bss _f.b
000000000000400c g     O __DATA,__common _a
0000000000004000 g     O __DATA,__data _b
0000000000003fa0 g     F __TEXT,__text _f
0000000000003fb0 g     F __TEXT,__text _g
0000000000000000         *UND* dyld_stub_binder
```

一眼就能看出他们的含义，不多说。

> 很有意思的一点：初始化为 0 的静态变量 `_f.b`（`static int b = 0`）在 `__bss` 段，而初始化为非零值的变量在 `__data` 段。`bss` 段存放未初始化或初始化为 0 的数据，默认会被初始化为 0，因此是等效的。
> `__common` 表示还未分配位置的数据（未初始化的全局变量，如全局 `int a = 0` 在某些编译器实现中被视为 COMMON 符号）。

- `*UND*` 表示需要重定位的符号（即在本模块中引用但未定义，需要在链接时从其他模块解析）

## 符号的分类

1. **全局符号**：非静态的全局函数和全局变量（如 `_b`、`_f`、`_g`），可被其他模块引用
2. **本地符号**：静态全局函数、静态全局变量和静态局部变量（如 `_f.a`、`_g.a`），只在当前编译单元内可见（小写 `l` 表示 local）
3. **外部引用**：在本模块中引用但在其他模块中定义的符号（`*UND*`）
