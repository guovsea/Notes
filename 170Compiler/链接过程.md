## 概念

### 目标文件 Object File

- 可重定位目标文件（Relocatable Object File）
- 可执行目标文件（Executable Object File）
- 共享目标文件（Shared Object File）

源码 --预处理/编译/汇编-> 可重定位目标文件 --链接-> 可执行目标文件/共享目标文件

目标文件具有特定的格式，在 Linux 上是 `ELF (Executable and Linkable Format)`，在 Windows 上是 `PE (Portable Executable)`，在 macOS 上是 `Mach-O`。

#### 可重定位目标文件（以 ELF 为例）

- ELF 头
- `.text` -- 编译后的机器代码
- `.rodata` -- 只读数据（如字符串常量、const 变量）
- `.data` -- 已初始化的全局/静态变量
- `.bss` -- 未初始化或初始化为 0 的全局/静态变量（不占文件空间）
- `.symtab` -- 符号表，存放定义和引用的符号
- `.rel.text` -- 代码段中需要重定位的位置
- `.rel.data` -- 数据段中需要重定位的位置
- `.debug` -- 调试信息
- `.line` -- 源代码行号映射
- `.strtab` -- 字符串表
- 节头部表（Section Header Table）

`.symtab` 存放了该目标文件中定义的符号以及引用的符号，即全局函数和全局变量。

`.rel.text` 中保存了需要被重定位的代码段。任何调用外部函数或者引用全局变量的指令都保存在这里，需要在链接时被重定位。`.rel` => relocation。

`.rel.data` 中保存了需要被重定位的 `.data` 段（如全局变量的初始化值是其他模块中定义的函数或变量的地址）。

## 链接的两个核心步骤

### 1. 符号解析（Symbol Resolution）

将每个模块中引用的符号与其定义关联起来。链接器维护一个符号表，依次扫描各模块，记录每个符号的定义和引用。详见 [[符号和符号表]]、[[链接多个模块时存在同名符号]]。

### 2. 重定位（Relocation）

将各模块的代码和数据合并到最终的地址空间中，并修改所有对外部符号的引用，使其指向正确的运行时地址。

## 链接属性

[[链接属性 Linkage]]

- 没有链接属性（No linkage）：局部变量
- 内部链接（Internal linkage）：`static` 修饰的全局变量/函数，只在本编译单元可见
- 外部链接（External linkage）：普通全局变量/函数，可被其他编译单元引用

## Linker's Symbol Rules

### Strong / Weak Symbols

- **强符号**：已初始化的全局变量、函数定义
- **弱符号**：未初始化的全局变量

详见 [[链接多个模块时存在同名符号]]

### Rules

1. Multiple strong symbols are not allowed（链接报错）
2. 一个强符号多个弱符号：选择强符号，其他弱符号引用强符号
3. 多个弱符号：pick an arbitrary one
   > 可以被 gcc `-fno-common` 覆盖，也就会报错
