#include <iostream>
#include <cmath>
using namespace std;

bool isPrime(int n) {
  
    // Check if n is 1 or 0
    if (n <= 1)
        return false;

    // Check if n is 2 or 3
    if (n == 2 || n == 3)
        return true;

    // Check whether n is divisible by 2 or 3
    if (n % 2 == 0 || n % 3 == 0)
        return false;
    // Check numbers of the form 6k ± 1 up to √n
    for (int i = 5; i *i<=n; i = i + 6)
        if (n % i == 0 || n % (i + 2) == 0)
            return false;

    return true;
}
int main() {
    int n = 7;  
  	if(isPrime(n)) cout << "true";
  	else cout<<"false";
    return 0;
}

## 1. 合数一定能分解为质因数

数学基本定理（算术基本定理）：

每个大于 1 的整数，都可以分解为质数的乘积：

$$
n = p_1 p_2 \cdots p_k
$$

推论：

如果 $n$ 是合数，则一定存在某个质数 $p$，使得：

$$
p \mid n
$$

---

## 2. 逆否命题（质数判定核心）

原命题：

如果 $n$ 是合数，则存在质数 $p$ 能整除 $n$

逆否命题：

如果 $n$ 不能被任何质数整除，则 $n$ 是质数：

$$
(\forall p \le n,\ p \ \text{prime},\ p \nmid n) \Rightarrow n \ \text{is prime}
$$

---

## 3. 试除范围可以缩小到 $\sqrt{n}$

关键性质：

如果 $n$ 是合数，则：

$$
n = a \cdot b \Rightarrow \min(a,b) \le \sqrt{n}
$$

因此只需要检查 $\sqrt{n}$ 以内的因子。

---

## 4. 质数难以枚举 → 使用超集代替

理想情况：

使用所有质数去试除。

但问题是：

- 生成质数集合本身需要筛法
- 成本较高

因此工程上使用“质数超集”替代质数集合。

常见超集：

- $\{2,3\} \cup \{6k \pm 1\}$

---

## 5. 6k±1 的来源（为什么合理）

任意整数可写为：

$$
n = 6k + r,\quad r \in \{0,1,2,3,4,5\}
$$

逐类分析：

- $6k$：能被 2,3 整除
- $6k+2$：偶数
- $6k+3$：能被 3 整除
- $6k+4$：偶数
- $6k+1$：可能为质数
- $6k+5 = 6(k+1)-1$：可能为质数

因此，只有：

$$
6k \pm 1
$$

可能不含 2 和 3 这两个小质因子。

---

## 6. 为什么用 6k±1 去做试除

试除法本质是：

检查是否存在质因子 $p$ 使得：

$$
p \mid n
$$

关键事实：

如果 $n$ 是合数，则一定存在质因子 $p \le \sqrt{n}$。

而所有质数（>3）必然落在 $6k \pm 1$ 中。

因此可以用该集合作为“质数候选超集”。

---

## 7. 完整判定逻辑

最终算法逻辑为：

1. 只要检查所有候选因子 $i$
2. 判断是否满足：
   $$
   n \bmod i = 0
   $$
3. 若存在则为合数，否则为质数

形式化表达：

$$
\forall i \in S,\ i \nmid n \Rightarrow n \ \text{prime}
$$

其中：

$$
S = \{2,3\} \cup \{6k \pm 1\}
$$

---

## 8. 核心总结

质数判断的本质是：

- 合数必然存在质因子
- 质因子一定 $\le \sqrt{n}$
- 质因子集合可以用结构化方式（如 6k±1）近似生成
- 试除法是在验证“是否存在质因子”

最终可以理解为：

> 用一个“质数候选集合”去覆盖所有可能的质因子，如果没有命中，则 $n$ 为质数。
