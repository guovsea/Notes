[三数之和
](https://leetcode.cn/problems/3sum/submissions/)

给你一个整数数组 nums ，判断是否存在三元组` [nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k `且 `j != k `，同时还满足 `nums[i] + nums[j] + nums[k] == 0 `。请

你返回所有和为 0 且不重复的三元组。

注意：答案中不可以包含重复的三元组。
## 思路

**排序 + 双指针**

不能包含重复的三元组，因此要去重，暴力方法是三重循环`i, j, k`，找到所有使得`nums[i] + nums[j] + nums[k] == 0` 的`i, j, k`组合。但是这样就会有很多重复的，对于一组`i, j, k`,那么就会有6种组合：`i , j, k`、`i, k, j`、`j, i, k` ……

只需要排序，就可以根据指针的范围确定`i, j, k`的大小关系，`i <= j <= k`，这样`i, j, k`就只有一种组合了

对于排序了的数组，可以有很多优化：

1. [i]+[i+1]+[i+2] > A，那么后面不可能有更小的组合能 = A

2. \[i] + [j] > A，说明 j 要左移，[i] + [j] < A，说明 i 要右移**双指针**

3. \[i] > A，那么等于 A 的数只会在 i 左边，[i] < A，只会在 i 右边**二分法**
4. 

```cpp
class Solution {
public:
vector<vector<int>> threeSum(vector<int>& nums) {
    sort(nums.begin(), nums.end());
    int N = nums.size();
    vector<vector<int>> ans;
    for (int i = 0; i < N; i++) {
        if (i > 0 && nums[i] == nums[i - 1]) {
            continue;
        }
        // 优化，如果[i]+[i+1]+[i+2] > 0,那么后面不可能有更小的组合能=0
        if (i < N - 2 && nums[i] + nums[i + 1] + nums[i + 2] > 0) {
            break;
        }
        // 转化为在一个有序数组中找一对数字和为target
        int target = - nums[i];
        int L = i + 1;
        int R = N - 1;
        while (L < R) {
            // ==target
            if (nums[L] + nums[R] == target) {
                ans.push_back(vector<int>({nums[i], nums[L], nums[R]}));
                // == targe是才需要去重
                // 1 2 2 2 3 遇到一串重复数时，L会指向重复数中最后一个
                while(L < R && nums[L] == nums[L + 1]) {
                    L++;
                }
                // R 会指向重复数中的第一个
                while (L < R && nums[R] == nums[R - 1]) {
                    R--;
                }
                // 继续指向下一组解
                L++;
                R--;
            } else if (nums[L] + nums[R] < target) {
                L++;
            } else {
                R--;
            }
        }
    }
    return ans;
}
};
```

**复杂度分析**

- 时间复杂度：O(n^2)，排序 O(n log n) + 双指针遍历 O(n^2)
- 空间复杂度：O(1)，不考虑结果数组
