## 1.确定本次调用的候选函数 candidate function

- 与被调函数同名
- 声明在调用点可见

## 2.确定可行函数 viable function

- 形参数量与实参数量相等 *如果函数形参有默认值，实参数量可以少于形参数量*
- 每个形参的类型与对应实参的类型相同或者能转换成形参的类型

**如果没有可行函数，编译将报错，error：no matching function for call to**

## 3. 寻找最佳匹配

形参类型与实参类型越接近，**类型匹配约精确**

如果**有且只有一个**函数满足：

- 该函数**每个形参与实参的匹配程度都不劣于**其他可行函数
- **至少有一个形参的匹配程度优于**其他可行函数

则匹配成功

**如果不止这样一个函数，编译将报错，call to 'f' is ambiguous**

> 调用重载函数时应尽量避免强制类型转换。如果需要，则说明设计的形参集合不合理


类型匹配的精确程度由高到低

## 匹配精度

1. **精确匹配**
   - 形参与实参的类型相同
   - 实参类型从数组或普通函数转换成相应的指针 *成员函数无法自动转换成指针：*[[成员函数指针]]
   - **向实参中添加顶层 const 或删除顶层 const** （是否向实参顶层对实参而言并没有意义，因为是值传递；**顶层 const 不影响函数匹配**）
2. **const 转换**（底层非 const 转换成底层 const，**不能反向转换**）实现的匹配
3. **整型提升**（整型提升：小正数提升为大整数，bool 、char 也算）实现的匹配
4. **算数类型转换**（整数->浮点数）实现的匹配
5. **类类型转换**（类定义了只接受一个参数的构造函数，则定义了参数类型到这个类类型的转换）

## 对一些转换的说明

*3/4 很难区分，大致根据“是否需要改变对底层 01 的解释”进行区分。一般人很少写这种屎一样的代码*

**整型提升和算数类型转换**

整型提升是指：小整数提升为大整数，这样的转换不会损失任何精度

`char(1)->short(2) -> int(4) -> long(4/8)`

*区别？对底层存储影响的程度不通。整型提升只需要添加几个 0 位；算数类型要完全改变数据的存储方式，或者说转换前后的两个类型对内存的解释方式不同*

**算数类型转换**

**除了整型提升以外**的其他算数类型之间的转换，比如 int->double


```cpp
void f(int arg) {
    std::cout << "short->int" << std::endl;
}
void f(unsigned short arg) { }
int main(void) {
    f(short(3));   // 输出 short->int
}
```
