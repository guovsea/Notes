在 C++ 中，虚函数（virtual function）是一个类的成员函数，可以在派生类中重写（override）。**在使用虚函数时，不应该使用默认参数**。这是因为：

 **默认参数会被静态绑定，虚函数在运行时进行动态绑定**。如果虚函数使用了默认参数，那么**编译器在编译期间会根据对象的静态类型来确定默认参数的值**，这个值不会根据对象的实际类型而改变。


举个例子：

`code 1`

```cpp
#include <iostream>

class Base {
public:
    virtual void foo(int x , int y = 5) {
        std::cout << "Base::foo(" << x << "," << y<< ")" << std::endl;
    }
};

class Derived : public Base {
public:
    virtual void foo(int x, int y = 3) override {
        std::cout << "Derived::foo(" << x << "," << y<< ")" << std::endl;
    }
};

int main() {
    Base* b = new Derived();
    b->foo(5); // Output: Derived::foo(5,5)
    b->foo(5, 2); // Output: Derived::foo(5,2)
    delete b;
    return 0;
}

```

可以看到：**被调用的函数是定义在对象的实际类型中的虚函数**这一动态特性还是成立的；但形参却被赋值为**对象的静态类型中声明的虚函数的默认值**

> 如果不依赖默认参数则没有任何影响（意味着会产生难以发现的 bug）

**函数形参默认的绑定发生在编译期间，而不是运行期间**
