## 概述

`unique_ptr` 和 `shared_ptr` 之间的区别是所有权语义不同：

- 内存对象只能被一个 `unique_ptr` 对象所指向
- 内存对象可以被多个 `shared_ptr` 对象所指向

最好用 `unique_ptr`，因为 `unique_ptr` 对象只有一个指针的大小（指向拥有的对象），`shared_ptr` 有两个指针的大小（指向拥有的对象，指向控制块）。


## 注意事项

1. **将智能指针声明为自动变量**。 （不要对智能指针本身使用 **`new`** 或 `malloc` 表达式。）
2. 智能指针只能管理堆对象，不能管理栈上对象
3. 不要用同一个裸指针构造多个智能指针
4. 不要直接用 `this` 指针构造智能指针（该情况可以被事项 3 涵盖）
5. 谨慎使用 get()
6. 对象一但被智能指针所管理，便不用再担心它的生命周期，只需要使用即可

### 智能指针不能管理栈上对象

```cpp
#include <iostream>
#include <memory>
struct A {
    ~A() { std::cout << "A 被销毁" << std::endl; }
};

int main() {
    {
        A a;
        std::shared_ptr<A> pa(&a);
    }// 段错误
}
```
### 不能用裸指针构造多个智能指针

如果直接用裸指针构造多个只能指针，那么每个智能指针都不知道其他智能指针的存在

```cpp
#include <iostream>
#include <memory>
struct A {
    ~A() { std::cout << "A 被销毁" << std::endl; }
};

int main() {
    {
        A *pa = new A;
        std::unique_ptr<A> p1(pa);
        std::unique_ptr<A> p2(pa);
    }
    std::cout << "into external scope\n";
}
```

### 不要直接用 `this` 指针构造智能指针

```cpp
#include <iostream>
#include <memory>
struct A {
    A() {}
    ~A() { std::cout << "A 被销毁" << std::endl; }
    std::unique_ptr<A> f() {
        return std::unique_ptr<A>(this);
    }
};

int main() {
    {
        A *pa = new A;
        auto p1 = pa->f();
        auto p2 = pa->f();
    } // 段错误
    std::cout << "into external scope\n";
}
```

### 谨慎使用 `get()`

1. 不要保存 `get()` 返回的裸指针    
   - 如果保存为裸指针，该指针会在某个位置变成空悬指针  
   - 如果保存为智能指针，则会被多次析构
2. 对 `get()` 返回的的裸指针使用 `delete` 
   会对一块内存 `delete` 两次
