**某些函数需要将其实参连同类型不变地转发给其他函数**，包括：实参类型是否是 const、实参是左值还是右值。

**使用 T&& 万能绑定**足以

```cpp
void f(int v1, int & v2) {
    std::cout << v1 << "  " << ++v2 << std::endl;
}
template<typename F, typename T1, typename T2>
void func1(F f, T1 t1, T2 t2) { 
    f(t2, t1);
}

template<typename F, typename T1, typename T2>
void func2(F f, T1 &&t1, T2 &&t2) {
    f(t2, t1);
}

int main(void) {
    int i = 3;
    func1(f, i, 42); // T1, T2 被推导为int
    std::cout << i << std::endl;
    int j = 3;
    func2(f, j, 42); // T1被推导为int&，t1的类型为int& &&折叠为int&，T2被推导为int, t2的类型为int&&
    std::cout << j  << std::endl;
}
```
output:
```text
42  4
3
42  4
4
```
但是传递给的函数接受的参数是右值引用参数时，虽然 T && t 最后的 t 的类型是 T &&，但 t 是一个变量，也是一个左值，无法将右值引用绑定到一个左值。

```cpp
template<typename F, typename T1, typename T2>
void func2(F f, T1 &&t1, T2 &&t2) {
    f(t2, t1); // rvalue reference to type 'int' cannot bind to lvalue of type 'int'G
}
// 不能将 v1 绑定到一个 t2，因为 t2 是一个变量
void g(int &&v1, int & v2) {
    std::cout << v1 << "  " << ++v2 << std::endl;
}
int main(void) {
    int i = 3;
    func2(g, i, 42);
}
```

## std::forward<>

`std::forward`需要显式指定模板实参。

可以抽象地认为：`std::forward<Type>` 会返回 `Type&&`

```cpp
template <typename Type> intermediary(Type && arg) {
    func(std::forward<Type>(arg));
}
```

首先`Type && arg`中的`Type`会保留传入`intermediary`中的实参的所有类型 (const 和非 const），`std::forward<Type>`会返回 `Type&&` 类型。

如果实参是左值引用，arg 为`Type &`类型（因为引用折叠），传入 func 时类型就为`Type&& &`再次引用折叠，传入 func的实际类型为`Type&`。

如果实参是右值引用，arg 为`Type &&`类型，传入 func 时类型就为`Type&& &&`，因为引用折叠，传入 func 的实际类型为`Type&&` 。

**语义理解：在传入的 arg 上再套一层 T&&，arg 为 T&时引用折叠为 T&，arg 为 T&&时引用折叠为 T&&**。

但也得要保证传入左值时 arg 为 T&，传入右值是 arg 为 T&&。因此 `T&& arg`和`std::forward<T>(arg)`都必不可少。


```cpp
template<typename F, typename T1, typename T2>
void func2(F f, T1 &&t1, T2 &&t2) {
    f(std::forward<T2>(t2), std::forward<T1>(t1));
}
void g(int &&v1, int & v2) {
    std::cout << v1 << "  " << ++v2 << std::endl;
}
int main(void) {
    int i = 3;
    func2(g, i, 42);
```
