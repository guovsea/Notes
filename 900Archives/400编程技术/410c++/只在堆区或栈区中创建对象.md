## 只在栈区中创建对象

将 `operator new` 声明为 `private`

[[new delete]]

```cpp
class A  
{  
private:  
    void* operator new(size_t t){}     // 注意函数的参数和返回值都是固定的  
    void operator delete(void* ptr){} // 重载了new就需要重载delete  
public:  
    A(){}  
    ~A(){}  
};
```
## 只在堆区中创建对象


将构造函数声明为 private，在静态函数 `create` 中创建对象并返回其指针/引用。

```cpp
class A{
public:
    static A* create() {
        return new A();
    }
    void destroy(){delete this;}
private:
    A(){}
}
```

改进：**使用智能指针**

```cpp
#include <memory>

// ! 错误代码
class A{
public:
   static ::std::shared_ptr<A> create() {
      return ::std::make_shared<A>();
   }
    void destroy(){delete this;}
private:
    A(){}
}
```


使用智能指针的正确版本

[[解决 make_shared 无法访问非 public 构造函数的问题]]

```cpp
class A
{
public:
    // 派生类指针隐式转换为基类指针
    static std::shared_ptr<A> create()
    {
        // make_shared_enabler 继承自 A，因此包含 A 类的所有信息
        struct make_shared_enabler : public A {};
        // 生成了指向 make_shared_enabler 的指针
        return std::make_shared<make_shared_enabler>();
    }

private:
    A() {}  
};
```

> 补充

除了将构造函数声明为 private 外，将析构函数声明为 private 也可以

原理：**编译器在为类对象分配栈空间时, 会检查类的构造函数和、析构函数的访问性，如果其中任何一个函数非 public，则编程器不会在栈空间上为类对象分配内存。**

```cpp
class A{
public:
    A(){}
    void destroy(){delete this;}
private:
    ~A(){}
}
```

存在问题：

1. 无法被继承。**继承的时候需要把析构函数设置为虚函数否则不能调用子函数的析构函数**，
2. new 一般会联想到 delete，并不会想到 destroy ，容易引起误解。
