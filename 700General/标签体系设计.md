# 标签体系设计

## 设计原则

- 标签用于**跨文件夹的横向索引**，不重复文件夹已提供的领域分组
- 采用 **3 个维度**，用前缀区分：领域主题（无前缀）、内容类型（`type/`）、活动用途（`act/`）
- 子标签统一使用**英文**，保持一致性
- 每篇笔记 1-3 个标签即可，避免过度标注

---

## 三维标签体系

| 维度 | 前缀 | 用途 | 示例 |
|------|------|------|------|
| **领域主题** | 无前缀 | 笔记属于什么知识领域 | `cpp/template`, `os/thread` |
| **内容类型** | `type/` | 这是什么类型的笔记 | `type/concept`, `type/howto` |
| **活动/用途** | `act/` | 笔记与什么活动相关 | `act/leetcode`, `act/project` |

## 维度 2：内容类型

| 标签 | 含义 |
|------|------|
| `type/concept` | 核心概念解释（如"什么是左值"） |
| `type/howto` | 实践操作指南（如"CMake配置Qt项目"） |
| `type/reference` | 资料/速查表（如"SQL语法"） |
| `type/snippet` | 代码片段 |
| `type/pitfall` | 踩坑/陷阱/常见错误 |
| `type/summary` | 知识汇总（如"知识点汇总"） |
| `type/analysis` | 深度分析/调研 |
| `type/notes` | 学习笔记 |

---

## 维度 3：活动/用途

| 标签 | 含义 |
|------|------|
| `act/leetcode` | LeetCode 题解 |
| `act/project` | 项目相关笔记 |
| `act/interview` | 面试相关 |
| `act/daily` | 日常记录 |

---

## 迁移对照表

现有标签需要改动的映射关系：

| 现有标签 | 建议改为 |
|---------|----------|
| `cpp/机制` | `cpp/basics` 或具体子类 |
| `cpp/语法` | `cpp/basics` |
| `cpp/技巧` | `cpp/basics` + `type/howto` |
| `cpp/概念` | `cpp/basics` + `type/concept` |
| `cpp/代码片段` | `cpp/basics` + `type/snippet` |
| `reference` | `type/reference` |
| `pitfall` | `type/pitfall` |
| `leetcode` | `act/leetcode` |
| `network` | `network/protocol`（或根据内容具体化） |
| `learning` | `type/notes` |
| `state/未开始` | 保留或用 `mastery` 字段替代 |
| `project/graduation` | `act/project` + `project/graduation` |
| `错误总结` | `type/pitfall` |
| `误区纠正` | `type/pitfall` |

## 标签示例

C++ 模板概念笔记：

```yaml
tags:
  - cpp/template
  - type/concept
  - cpp/meta
mastery: 3
```

LeetCode 动态规划题解：

```yaml
tags:
  - algorithm/dp
  - act/leetcode
  - type/howto
```

MySQL 索引踩坑：

```yaml
tags:
  - database/index
  - type/pitfall
  - database/innodb
```
