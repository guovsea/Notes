## 相关

- [[MVC 模式]] -- Qt Model/View 架构是界面数据联动的标准方案
- [[信号和槽]] -- 数据变更通过信号通知界面更新

## 核心思路

以数据为中心进行编程，而非以界面操作为中心：

1. **将用户操作转化为对数据的修改**：界面控件（按钮、输入框等）的信号连接到数据修改逻辑
2. **数据修改后刷新界面**：数据模型发出变更信号，界面监听信号并更新显示

## 实现模式

```
用户操作 → 信号 → 修改数据 → 数据变更信号 → 界面更新
```

### 方式一：手动信号槽联动

```cpp
// 界面 → 数据
connect(ui->lineEdit, &QLineEdit::textChanged,
        this, &MyClass::onDataChanged);

// 数据 → 界面
void MyClass::updateDisplay(const QString &text) {
    ui->label->setText(text);
}
```

### 方式二：使用 Qt Model/View

利用 [[MVC 模式]]，数据模型自动通知视图更新，无需手动管理信号连接。

```cpp
QStringModel *model = new QStringListModel(this);
ui->listView->setModel(model);

// 修改数据后，视图自动更新
model->setStringList(newData);
```

### 方式三：数据绑定（Qt 5.15+ / Qt 6）

使用 `QBindable` 或第三方绑定库实现声明式的数据-界面绑定。
