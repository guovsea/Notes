## MVC

在交互式软件出现的早期，所有用户的输入：键盘、鼠标都由一个同一的对象控制 —— Controller。

在这种模式中，Controller 负责响应用户输入，更新 Model 和 View。

View 可以直接和 Model 通信。

MVC 是一种设计思想，并没有明显的设计模式和接口设计。

## MVP

整体思想：所有的复杂逻辑都在 Presenter 中，View 只简单的获取数据和设置数据。

接口设计：依赖于接口而非实现，通过依赖注入的方式解耦。

代码示例：[[MvpExample]]

## MVVM

整体思想：数据驱动，只需要通过 Model 构建 View Model （仍然是数据模型），框架实现了 View Model 和 View 的交互。

代表：声明式 UI 框架，比如 QML。
