需要区别的变量：

| Variable                 | Info                                                                                      |
| ------------------------ | ----------------------------------------------------------------------------------------- |
| CMAKE_SOURCE_DIR         | **The root source directory**                                                             |
| CMAKE_CURRENT_SOURCE_DIR | The source directory being processed.                                                     |
| PROJECT_SOURCE_DIR       | The source directory of the current project(**current sub-project**).                     |
| CMAKE_BINARY_DIR         | The root binary / build directory. This is the directory where you ran the cmake command. |
| CMAKE_CURRENT_BINARY_DIR | The build directory being processed.                                                      |
| PROJECT_BINARY_DIR       | The build directory for the current project(current sub-project).                         |

> CMake 项目有子文件夹时，子文件夹中的 `CMakeLists.txt` 中可能没有使用 `project()` 定义子项目，因此在该 `CMakeLists.txt` 中，只有 `CMAKE_CURRENT_SOURCE_DIR`，没有  `PROJECT_SOURCE_DIR`

`CMAKE_CURRENT_SOURCE_DIR` vs `PROJECT_SOURCE_DIR` 
 
> Let's say you have a directory called `Inner1` containing a CMakeLists.txt file without calling `project` inside of it. Then `PROJECT_SOURCE_DIR` is not set with `Inner1` 's dir path, but the `CMAKE_CURRENT_SOURCE_DIR` is set when being processed.
