## 连接层

负责和客户端进行通信，提供安全认证和权限验证。

## 服务层

- SQL 接口
- SQL 解析，SQL 的分析和优化，部分内置函数的执行
- 查询优化，确定表的查询的顺序，是否利用索引
- 查询缓存，加快 SELECT 语句的执行

## 存储引擎层

[[存储引擎]]
- 存储引擎真正的负责了 MySQL 中数据的存储和提取，服务器通过 API 和存储引擎进行通信。
- 不同的存储引擎具有不同的功能，可以根据自己的需要来选取合适的存储引擎。
- 数据库中的索引是在存储引擎层实现的。
- MySQL 采用插件式存储引擎架构，将查询处理和其他任务分离。这种架构可以根据业务的需求和实际需要选择合适的存储引擎。
## 数据存储层

数据存储层， 主要是将数据(如: redolog、undolog、数据、索引、二进制日志、错误日志、查询 日志、慢查询日志等)存储在文件系统之上，并完成与存储引擎的交互。
