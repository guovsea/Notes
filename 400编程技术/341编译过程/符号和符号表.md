
#### 符号和符号表

局部非静态变量没有符号，因此也可以说它具有 [[链接属性 Linkage#No linkage | No linkage]] 的链接属性。

静态局部变量存在符号，并且在.data 或.bss 段中。


## objdump 查看符号表

objdump -t

## 源文件

```cpp
int a = 0;
int b = 1;
int f() {
    static int a = 1;
    static int b = 0;
    return b;
}
int g() {
    static int a = 1;
    return 0;
}
```

编译为动态库

```
gcc mine.cpp -o libmine.so -shared -fPIC
```

## 符号表

```
objdump -t libmine.so
```

```shell
➜  staticLocalVar objdump -t libmine.so

libmine.so:     file format Mach-O 64-bit x86-64

SYMBOL TABLE:
0000000000004004 l     O __DATA,__data _f.a
0000000000004008 l     O __DATA,__data _g.a
0000000000004010 l     O __DATA,__bss _f.b
000000000000400c g     O __DATA,__common _a
0000000000004000 g     O __DATA,__data _b
0000000000003fa0 g     F __TEXT,__text _f
0000000000003fb0 g     F __TEXT,__text _g
0000000000000000         *UND* dyld_stub_binder
```

一眼就能看出他们的含义，不比多少

  > 很有意思的一点：始化为 0 的变量_g.b 在 `___bss` 段，而其他的变量在 `__data` 段。`bss` 段存放为初始化的数据，默认会被初始化为 0，因此是等效的。
  > `__common` 还未分配位置的数据，
  
- `*UND*` 表示需要重定位的符号
