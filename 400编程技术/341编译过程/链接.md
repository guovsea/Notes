## 概念

### 目标文件 Object File

- 可重定位目标文件
- 可执行目标文件
- 共享目标文件

源码  --预处理编译汇编-> 可重定位目标文件 --链接-> 可执行目标文件/共享目标文件

目标文件具有特定的格式，在 Linux 上是 `ELF(Executable and Linkable File)` ，在 windows 上是 `PE(Portable Executable)`
 

#### 可重定位目标文件

有 ELF、[[PE Portable Executable]]、Mach-O（OS-X）

ELF 为例

- ELF 头
- .text
- .rodata
- .data
- .bss
- .symtab
- .rel.text
- .rel.data
- .debug
- .line
- .strtab
- 节头部表

.symtab 存放了该目标文件中定义符号以及引用的符号，即全局函数和全局变量

.rel.text 需要被重定位的代码段。任何调用外部函数或者引用全局变量的指令都保存在这里，需要在链接时被重定位。.rel => relocation

.rel.data 需要被重定位的 .data 段

## Linker's Symbol Rules

### Strong / Weak Symbols

### Rules

1. Multiple strong symbols are not allowed
2. 一个强符号多个弱符号：处理强符号，其他弱符号引用强符号
3. 多个弱符号：pick an arbitrary one
   > 可以被 gcc -fno-common 覆盖，也就会报错
