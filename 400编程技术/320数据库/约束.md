**Create Table Constraints**

## 列约束

| 约束名称                  | 描述                           | 备注                                                     |
| ------------------------- | ------------------------------ | -------------------------------------------------------- |
| **NOT NULL**              | 非空                           |                                                          |
| **UNIQUE**                | 取值不允许重复                 |                                                          |
| **PRIMARY KEY**           | 主键                           | NOT NULL 且 UNIQUE                                       |
| **[[#FOREIGN KEY]]**      | 外键                           | [CONSTRAINT] [外键名称] FOREIGN KEY (外键字段名) REFERENCES 主表 (主表列名) ON 动作|
| **CHECK**(8.0.16版本之后) | 逻辑表达式约束                 | CHECK(条件表达式)                                        |
| **DEFAULT**               | 默认值（缺省值）               | DEFAULT 默认值 *没有=*                                        |
| **AUTO_INCREMENT**        | **自增长**的字段必须定义为主键 | AUTO_INCREMENT=起始值，默认 1                            |

创建表的时候列约束有两种表示方式

1. 在每个列定义后面声明 
2. 在所有列后面集中声明
3. 只能在列定义后面 NOT NULL，DEFAULT

   集中声明约束的时候，**每个约束条件后面都要有逗号**
**示例**
```sql
CREATE TABLE tb_user(  
id int AUTO_INCREMENT PRIMARY KEY COMMENT 'ID唯一标识', 
name varchar(10) NOT NULL UNIQUE COMMENT '姓名' ,  
age int check (age > 0 && age <= 120) COMMENT '年龄' , status char(1) default '1' COMMENT '状态',  
gender char(1) COMMENT '性别'
);
```

## 列级约束示例

### 1. NOT NULL

```sql
CREATE TABLE IF NOT EXISTS newauthor(
aut_id varchar(8) NOT NULL,    
aut_name varchar(50) NOT NULL,   
country varchar(25) NOT NULL,

```

### CHECK

- #### 值约束
```sql
CREATE TABLE IF NOT EXISTS newbook_mast (
no_page decimal(5,0) 
CHECK(no_page>0) , -- check constraint             
book_price decimal(8,2) ,                          
);
```

- #### 取值集合约束

```sql
CREATE TABLE IF NOT EXISTS newauthor(
country varchar(25) NOT NULL CHECK (country IN ('USA','UK','India')), 
);
```

- #### 模糊取值约束

```sql
CREATE TABLE IF NOT EXISTS newbook_mast(
dt_of_pub date CHECK (dt_of_pub LIKE '--/--/----'), 
);
```

- #### 带逻辑操作的约束

CHECK 的条件为真才允插入
```sql
CREATE TABLE IF NOT EXISTS newpublisher(
pub_id varchar(8) ,
pub_name varchar(50),
pub_city varchar(25) ,          
country varchar(25) ,
country_office varchar(25) ,
no_of_branch int(3),          
estd date 
CHECK ((country='India' AND pub_city='Mumbai') 
OR (country='India' AND pub_city='New Delhi')) , 
PRIMARY KEY (pub_id)
);
```

### UNIQUE

- #### 实现方法1（表的定义最后施加）

```sql
CREATE TABLE IF NOT EXISTS 
newauthor(aut_id varchar(8) NOT NULL ,
aut_name varchar(50) NOT NULL,
country varchar(25) NOT NULL,
home_city varchar(25) NOT NULL, 
UNIQUE (aut_id)
); 
```

- #### 实现方法2（字段定义的最后施加）
    

```sql
CREATE TABLE IF NOT EXISTS newauthor(
aut_id varchar(8) NOT NULL UNIQUE ,
aut_name varchar(50) NOT NULL,
country varchar(25) NOT NULL,
home_city varchar(25) NOT NULL
);
```

### FOREIGN KEY

#### 语法

```sql
CREATE TABLE 表名(  
字段名 数据类型,  
...，
[CONSTRAINT] [外键名称] FOREIGN KEY (外键字段名) REFERENCES 主表 (主表列名)
);

ALTER TABLE 表名 ADD CONSTRAINT [外键名称] FOREIGN KEY(外键字段) REFERENCES (主表列名)

ALTER TABLE 表名 DROP FOREIGN KEY 外键名称;
```

#### 外键动作

设置外键约束的本质就是在设置外键动作**外键动作 (referential action)**

语法 ：`[CONSTRAINT 外键名]  FOREIGN KEY (列名) REFERENCES 表名(列名) ON [触发 UPDATE/DELETE] [动作]`
```sql
FOREIGN KEY(ord_no,book_id) REFERENCES neworder(ord_no,book_id)
ON UPDATE CASCADE ON DELETE CASCADE,  -- 同时更新同时删除
```
常见动作组合

- ON UPDATE CASCADE ON DELETE RESTRICT *mysql 默认外键动作**
  
  父表（被引用的是父表）更新时子表也更新，父表删除时如果子表有匹配的项，删除失败；

- ON UPDATE CASCADE ON DELETE CASCADE
  
  父表更新时子表也更新，父表删除时子表匹配的项也删除。


五种外键动作

- CASCADE：当我们从父表中删除或更新任何行时，子表中匹配行的值将被自动删除或更新。
- SET NULL：当我们从父表中删除或更新任何行时，子表中的外键列的值会被设置为NULL。
- RESTRICT：当我们从父表中删除或更新在引用（子）表中具有匹配行的任何行时，MySQL 不允许删除或更新父表中的行。
- NO ACTION：类似于RESTRICT。但有一个不同之处，它会在尝试修改表后检查引用完整性。
- SET DEFAULT：MySQL 解析器能识别此橾作。但是，InnoDB 和 NDB 表都拒绝此橾作。

#### 为单个字段创建外键
```sql
CREATE TABLE IF NOT EXISTS newbook_mast (
book_id varchar(15) NOT NULL PRIMARY KEY,
book_name varchar(50)  ,
isbn_no varchar(15)  NOT NULL  ,
cate_id varchar(8)  , 
aut_id varchar(8) , 
pub_id varchar(8) ,          
dt_of_pub date ,
pub_lang varchar(15) ,
no_page decimal(5,0) ,         
book_price decimal(8,2) ,
FOREIGN KEY (aut_id) REFERENCES newauthor(aut_id)
);
```

- #### 为多个字段创建外键
```sql
CREATE TABLE IF NOT EXISTS newpurchase(
invoice_no varchar(12) NOT NULL UNIQUE PRIMARY KEY,
invoice_dt date ,

ord_date date ,
receive_dt date ,
book_id varchar(8) ,

pub_lang varchar(8) ,

total_cost decimal(12,2) ,
INDEX (ord_no,book_id),
FOREIGN KEY(ord_no,book_id) REFERENCES  neworder(ord_no,book_id),   
INDEX (cate_id),
FOREIGN KEY(cate_id) REFERENCES  category(cate_id)
);
```

- #### 创建来自不同表的外键

- #### 外键动作的设置
    
##### 1. UPDATE CASCADE（级联更新）和DELETE RESTRICT（限制删除）

```sql
CREATE TABLE IF NOT EXISTS newpurchase(
invoice_no varchar(12) NOT NULL UNIQUE PRIMARY KEY,     
invoice_dt date ,  
ord_no varchar(25) ,    
ord_date date ,    
receive_dt date ,      
book_id varchar(8) , 
book_name varchar(50) ,
pub_lang varchar(8) , 
cate_id varchar(8) , 
receive_qty int(5) , 
purch_price decimal(12,2) ,
total_cost decimal(12,2) , 
INDEX (ord_no,book_id),
FOREIGN KEY(ord_no,book_id) REFERENCES         
neworder(ord_no,book_id)
ON UPDATE CASCADE ON DELETE RESTRICT,  
INDEX (cate_id),
FOREIGN KEY(cate_id) REFERENCES category(cate_id)
);
```

##### 2. UPDATE CASCADE（级联更新）和DELETE SET NULL（删除置空）

```sql
CREATE TABLE IF NOT EXISTS newpurchase (
invoice_no varchar(12) NOT NULL UNIQUE PRIMARY KEY,       
invoice_dt date , 
ord_no varchar(25) ,
ord_date date ,
receive_dt date ,      
book_id varchar(8) ,
book_name varchar(50) ,
pub_lang varchar(8) ,     
cate_id varchar(8) , 
receive_qty int(5) , 
purch_price decimal(12,2) ,   
total_cost decimal(12,2) , 
INDEX (ord_no,book_id),
FOREIGN KEY(ord_no,book_id) REFERENCES neworder
(ord_no,book_id)
ON UPDATE CASCADE ON DELETE SET NULL,
INDEX (cate_id),
FOREIGN KEY(cate_id) REFERENCES category(cate_id)
);
```

##### 3. UPDATE CASCADE（级联更新）和DELETE NO ACTION（删除不操作）

```sql
CREATE TABLE IF NOT EXISTS newpurchase (
invoice_no varchar(12) NOT NULL UNIQUE PRIMARY KEY,        
invoice_dt date ,
ord_no varchar(25) ,
ord_date date ,
receive_dt date ,       
book_id varchar(8) ,
book_name varchar(50) , 
pub_lang varchar(8) ,
cate_id varchar(8) , 
receive_qty int(5) , 
purch_price decimal(12,2) ,
total_cost decimal(12,2) ,
INDEX (ord_no,book_id),
FOREIGN KEY(ord_no,book_id) REFERENCES         
neworder(ord_no,book_id)
ON UPDATE CASCADE ON DELETE NO ACTION,       
INDEX (cate_id),

);
```
