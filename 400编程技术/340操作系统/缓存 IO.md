## 缓存 IO 思想

缓存 I/O（Input/Output）是一种计算机数据存储和访问的优化技术，它通过将数据暂时存储在快速访问的存储介质中，以减少对慢速存储设备的频繁访问，从而提高系统性能。缓存 I/O 的主要思想是在内存或其他高速存储设备中维护一个副本或缓存数据，以避免每次数据访问都需要从较慢的存储介质（通常是硬盘或网络存储）中读取或写入数据。

缓存 I/O 可以出现在各种层次和场景中，包括：

1. **硬件缓存：** 计算机系统通常具有硬件级别的缓存，如 CPU 缓存和磁盘缓存。这些缓存用于加速数据访问，减少对主存或磁盘的访问延迟。
2. **操作系统缓存：** 操作系统可以维护文件系统缓存，将最近访问的文件数据存储在内存中，以加快文件读取操作。
3. **应用程序级别缓存：** 许多应用程序也使用缓存来提高性能。例如，Web 浏览器可以缓存已下载的网页和图像，以加快再次访问相同网页时的加载速度。
4. **数据库缓存：** 数据库管理系统通常具有查询结果缓存，将查询结果存储在内存中，以减少相同查询的执行时间。
5. **分布式系统缓存：** 在分布式系统中，缓存可以用来减少对远程服务器或存储的请求，从而降低网络延迟和减少服务器负载。
6. **代理服务器缓存：** 代理服务器可以缓存 Web 内容，以减少从源服务器获取数据的次数，提高访问速度。
  
## Linux
在 Linux 中，对于一次 IO 访问（以 read 举例），数据会先被拷贝到操作系统内核的缓冲区中，然后才会从操作系统内核的缓冲区拷贝到应用程序的地址空间。所以说，当一个 read 操作发生时，它会经历两个阶段：

1. 第一阶段：等待数据准备 (Waiting for the data to be ready)。  
2. 第二阶段：将数据从内核拷贝到进程中 (Copying the data from the kernel to the process)。

## 缓冲区
### 用户空间缓冲区

用户空间缓冲区本质上就是在用户空间中的一块内存，在使用`fork()`创建子进程的时候会被复制到子进程的内存中。`printf`和`std::cout`都有缓冲区。

#### 自定义缓冲

自己在程序中定制的缓冲，如`char buf[4096 * 1024]`
#### 标准 I/O 缓冲

**标准 I/O 缓冲是指标准库函数（如 C 语言中的`printf`、`scanf`、`fread`、`fwrite`等，以及 C++中的`cin`和`cout`等）在进行输入和输出操作时使用的内存缓冲区。** 用于暂时存储待写入文件的数据（输出缓冲）或待从文件中读取的数据（输入缓冲），从而减少调用系统调用的次数。**标准 I/O 是由 C/C++标准库维护的**

标准I/O缓冲通常有以下几种类型：

1. **全缓冲（Fully Buffered）**：在全缓冲模式下，数据会在内存缓冲区中积累，直到缓冲区满了，或者直到显式地要求将缓冲区的内容写入文件。这种模式通常用于磁盘文件，以减少与磁盘的交互次数。
2. **行缓冲（Line Buffered）**：在行缓冲模式下，数据会在内存缓冲区中积累，直到遇到换行符（`\n`），然后缓冲区的内容会被写入文件或提供给程序。这种模式通常用于终端设备，以实现逐行的输入和输出。
3. **无缓冲（Unbuffered）**：在无缓冲模式下，数据会立即写入文件或从文件读取，而不会在内存中积累。这种模式通常用于特殊的文件或设备，或者在需要立即写入或读取数据时。

标准 I/O 是指用于输入和输出操作的抽象接口，并不是指`STDIN_FIFO、STDOUT_FIFO`（这些是文件描述符）。C 标准库会使用标准库函数`isatty`检查文件描述符是否对应于终端设备，然后选择缓冲模式。（不重要，涉及内部实现，只是解答我的疑惑）

**I/O 缓冲在用户空间中，因此 fork () 时会被复制到子进程中**
```c++
#include <iostream>
#include <unistd.h>
int main(int argc, char const *argv[])
{
    pid_t pid;
    std::cout << "berore fork\n";
    // std::cout << "berore fork" << std::endl;
    if ((pid = fork()) == -1) {
       perror("fork error");
       exit(-1);
    } else if (pid == 0) {
       std::cout << "chirld ID = " << getpid() << std::endl;
    } else {
       std::cout << "parent ID = " << getpid() << std::endl;
    }
    sleep(1);
    return 0;
}
```
上述代码的结果：

- 不使用 `std:: endl` ，将输出到终端设备 **行缓冲**，遇到`\n`就将标准 I/O 缓冲刷新。
  >*刷新* ： 调用系统调用`write`，将标准 I/O 缓冲区传递到内核缓冲区，并清空。
```shell
guo@ubuntu:~/lean$ ./a.out
berore fork
parent ID = 58537
chirld ID = 58538
```
- 不使用 `std:: endl` ，将 `STDIO` 重定向到文件 **全缓冲**，遇到，遇到`\n`没有刷新标准 I/O 缓冲，而是在等标准 I/O 缓冲满。
```shell
guo@ubuntu:~/lean$ ./a.out > output
guo@ubuntu:~/lean$ cat output
berore fork
parent ID = 58570
berore fork
chirld ID = 58571
```
- 使用 `std:: endl` **强制**刷新标准 I/O 缓冲
```shell
guo@ubuntu:~/lean$ ./a.out > output
guo@ubuntu:~/lean$ cat output
berore fork
parent ID = 58657
chirld ID = 58658
```
### 内核缓冲

内核缓冲区是内核中的一块内存区域，由操作系统管理。当使用`write`系统调用时，会现将用户空间缓冲区中的内容拷贝到内核缓冲区，然后**由操作系统决定**真正刷入磁盘中的时机。因此这是异步的，但是可以使用`fflush`强制让操作系统将内容刷入磁盘。


在 Linux 的缓存 IO 机制中，操作系统会将 IO 的数据缓存在文件系统的页缓存（ page cache ）中，也就是说，数据会先被拷贝到操作系统内核的缓冲区中，然后才会从操作系统内核的缓冲区拷贝到应用程序的地址空间。
