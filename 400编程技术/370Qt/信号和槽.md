
## 信号和槽的本质

在 QT 中信号是由**对象**发出的，对象内部可以对事件进行检测

槽函数是一类特殊的功能函数，所有者是某个**对象**，槽函数也可以作为普通函数使用。

## 信号和槽的连接方式

### Qt4

`Qt 4.x`：使用字符串封装信号和槽
```cpp
// Qt4  
[static] QMetaObject::Connection QObject::connect(
    const QObject *sender, const char *signal, 
    const QObject *receiver, const char *method, 
    Qt::ConnectionType type = Qt::AutoConnection);

connect(const QObject *sender,SIGNAL(信号函数名(参数1, 参数2, ...)),
        const QObject *receiver,SLOT(槽函数名(参数1, 参数2, ...)));

```

**Qt4 中声明槽函数必须要使用 slots 关键字, 不能省略。**
```cpp
connect(ui->pushButton, SIGNAL(clicked()), this, SLOT(testPushBtn()));

```
**运行时**才输出：
```text
QObject::connect: No such slot MainWindow::testPushBtn() in [../begin/mainwindow.cpp:23](../begin/mainwindow.cpp:23)

QObject::connect: (sender name: 'pushButton')

QObject::connect: (receiver name: 'MainWindow')
```

```cpp
class MainWindow : public QMainWindow
{
    Q_OBJECT

public:
    MainWindow(QWidget *parent = nullptr);
    void test(int);
    ~MainWindow();
public slots:        // 必须加上 slots:
    void testPushBtn();
signals:
    void test();

private:
    Ui::MainWindow *ui;
};
```
### Qt5

`Qt 5.x`: 使用函数指针作为信号和槽


**connect 函数声明**
```cpp

// Qt5
QMetaObject::Connection QObject::connect(
    	const QObject *sender, PointerToMemberFunction signal, 
        const QObject *receiver, PointerToMemberFunction method, 
		Qt::ConnectionType type = Qt::AutoConnection);

// 信号和槽函数也就是第2,4个参数传递的是地址, 编译器在编译过程中会对数据的正确性进行检测
connect(const QObject *sender, &QObject::signal, 
        const QObject *receiver, &QObject::method);


```

## 自定义信号和槽

### 自定义信号

1. **继承自 QObject**
2. **类内有宏：Q_OBJECT**
3. **返回值必须是 void 类型**
4. **信号需要使用 signal 关键字修饰**（类似于 public）
5. **信号只需要声明，不行要定义（实现）**
6. **信号参数可以随意指定，支持重载**

在程序中发送自定义信号的本质就是**调用信号函数**

- 习惯在调用信号函数前加“关键字”：emit
- emit 只是显氏声明在发送信号，没有其他作用。（emit 是宏定义，且为空：`#define emit `）
- 为了提高可读性

```cpp
class MyClass : public QObject
{
    Q_OBJECT;
signals:
    void s1();
    void s2(int);
}
```
### 自定义槽函数

要求：

1. **返回值必须是 void 类型**
2. **槽函数支持重载**（槽函数就是普通函数）
3. **槽函数的参数要和信号中的参数类型一一对应**。因为槽函数的参数是用来接收信号传递的数据的, 信号传递的数据就是信号的参数。
> 信号的参数数量可以大于等于槽函数的参数数量，信号中剩余的信号参数会被忽略。


槽函数**可以**使用关键字进行声明: slots (Qt5 中 slots 可以省略不写)

```cpp
public slots:
private slots:     // 这样的槽函数不能在类外部被调用
protected slots: 
```

Qt 中槽函数的类型是多样的

Qt 中的槽函数可以是类的成员函数、全局函数、静态函数、Lambda 表达式等可调用对象。

*connect 函数第三个参数要求传入一个`*recv`，recv 对象和槽函数可以没有任何关联，槽函数实际上可以传入任意可调用对象*

```cpp
#include "mainwindow.h"
#include "ui_mainwindow.h"
#include "QPushButton"
struct Callable {
    void operator()() {
        qDebug() << "call a Callable obj";
    }
};

MainWindow::MainWindow(QWidget *parent)
    : QMainWindow(parent)
    , ui(new Ui::MainWindow)
{
    ui->setupUi(this);
    // 成员函数
    connect(ui->pushButton, &QPushButton::clicked, this, &MainWindow::testPushBtn);
    // lambda 表达式
    connect(ui->pushButton, &QPushButton::clicked, this, [](){
        qDebug() << "lambda";
    });
    // 可调用对象 （注意，不能传入可调用对象的地址）
    Callable call;
    connect(ui->pushButton, &QPushButton::clicked, this,
            call);
}

void MainWindow::testPushBtn() {
    qDebug() << "btn clicked";
}
MainWindow::~MainWindow()
{
    delete ui;
}
```

## 信号槽拓展

一个信号可以绑定多个槽函数，出发多个动作；

- 需要写多个`connect`
- 信号产生后，`Qt4.x` 的槽函数执行顺序是随机的；`Qt5.x` 的槽函数会按照 connect 顺序执行

一个槽函数可以被多个信号绑定

信号接收者还可以在槽函数中继续发出信号

信号可以直接绑定信号

- 比如当 QPushButton 产生 clinked 信号时，就不再需要写一个槽函数发送自定义信号了，而是可以直接姜 clinked 信号绑定到自定义的信号。换句话说就省去了中间使用槽函数转发的步骤。

## 重载信号和槽

**如果只是用函数名会产生歧义，因为重载函数的函数名都相同**

解决办法：

1. Qt4 
```cpp
// Qt4处理方式
connect(&m, SIGNAL(eat()), &m, SLOT(hungury()));
// 直接在参数列表中指定参数类型
connect(&m, SIGNAL(eat(QString)), &m, SLOT(hungury(QString)));
```
2. Qt5

定义函数指针类型的变量，再姜这个变量传入 connect。在定义函数指针类型时，会消除歧义。

```cpp
// 定义一个函数指针类型的变量，具体指明特定函数
void (Me::*mysignal)(QString) = &Me::eat;
// void (Me::*mysignal2)() = &Me::eat; // 是两个不同的函数(编译器能够知道指针指向的到底哪个函数)
void (Me::*myslot)(QString) = &Me::hungury;
// 传入函数指针变量
connect(&m, mysignal, &m, myslot);
```

定义重载函数的指针
```cpp
void func(int x) {
    std::cout << "func(int)" << std::endl;
}
void func(void) {
    std::cout << "func()" << std::endl;
}
int main() {  
    void (*p1)(int) = func; // 编译器能够知道指针指向的到底哪个函数
    void (*p2)() = func;
    p1(3); 
    p2();
}
// 输出
// func(int)
// func()
```
