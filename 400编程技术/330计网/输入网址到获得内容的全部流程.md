## 基础检查
1. 浏览器检查网址的正确性
2. 检查本地是否存在缓存,如果存在缓存则直接加载缓存
## 获取 ip 地址
3. 检查游览器 dns 缓存中是否存在dns 信息, 检查主机的 hosts 文件,如果存在缓存则能直接获取其 ip 地址
4. 向本地 dns 服务器发送 dns 请求, 本地 dns 也存在缓存策略
5. 如果本地 dns 服务器缓存中不存在的话,则以迭代的方式分别向:dns根服务器, dns顶级域服务器, dns 权威服务器获取下一层级的 ip 地址,最终查询到 网址真是的 ip 地址
## 封装
6. 逐层封装 http 报文, http 协议是使用的 tcp 协议, http 报文在运输层会被加上 tcp 头部,然后网络层在加上 ip 头部,最后在数据链路层被加上帧头和帧尾
## 发送
> 发送过程可分为局域网内和局域网间
7. 局域网内的发送端查看目的 ip 地址是否存在与 mac 转发表中,如果存在则在帧头步添加目标 mac 地址并从相应的端口发出
> 如果目标 ip 地址不在转发表中, 则使用 ARP 协议广播改 ip 地址, 包含改 ip 地址的接收端做出回应, 回应它的 mac 地址
8. 局域网间需要通过路由器进行发送, 路由器中都有一个转发表, 记录这每个 ip 地址所对应的端口,当接收到某个package 时,会根据它的 ip 地址选择某个端口进行发送
## 解封装
9. 服务器端在接受到数据后会逐层解封装, 最终将 http 报文发送给上层应用,上层应用根据 http 报文做出响应, 将客户端请求的网页发送到客户端,然后在通过浏览器对网页内容进行渲染
