非对称加密(*公开密钥加密*)的核心原理:
**公钥用于数据加密. 公钥加密的数据只能用对应的私钥解密**

1. 消息接收方准备好公钥和私钥
2. 私钥接收方自己留存、公钥发布给消息发送方
3. 消息发送方使用接收方公钥对消息进行加密
4. 消息接收方用自己的私钥对消息解密

形象理解:
一个信箱有两个口:收信口和取信口。给我发送信封的所有人都有收信口的钥匙,可以把信放到收信口后在锁上。我想要取信的时候只能用取信口的钥匙取信。在这里,将信封放入收信口并上锁就是利用公钥对数据进行加密，用钥匙将取信口打开并拿出信就是利用私钥对数据进行解密


RSA 算法
1. 密钥生成：
- 随机选择两个大素数p和q，它们通常是几百位或更长的大素数。
- 计算n = p * q，n为公共模数。
- 计算欧拉函数φ(n) = (p - 1) * (q - 1)。
- 选择一个与φ(n)互质的整数e（1 < e < φ(n)）作为公钥的指数。
- 计算e的模反元素d，使得 (d * e) % φ(n) = 1。d为私钥的指数。

2. 加密：
- 将明文消息转换为一个数字M，确保M < n。
- 使用公钥中的模数n和指数e，计算密文C = M^e mod n。

3. 解密：
- 使用私钥中的模数n和指数d，计算明文消息 M = C^d mod n。

RSA算法的安全性基于大整数分解问题的困难性。解密需要找到p和q的乘积n的质因子分解。目前没有已知的有效算法可以在合理的时间内解决大整数分解问题，因此，只要密钥的位数足够大，RSA算法被认为是安全的。

安全性:
传播: n, e, C 
解密: n, d, C
因此,要解密必须通过 n 和 e 得到 d ,要得到 d 就必须知道 φ(n) ,就必须知道 p 和 q, 因此也就必须对 n 进行质因数分解
