### 1. 背景

路由是[指分组从源到目的地时，决定端到端路径的网络范围的进程](James F.Kurose,Keith W.Ross 著.陈鸣译．《计算机网络》．机械工业出版社．2009)。路由器（router）在 OSI 参考模型中的网络层（Network）中工作，负责在不同网络之间传输数据包，并确保数据包能够正确到达目标地址。

静态路由通过手动配置路由器的路由表来确定数据包的传输路径，路由表由若干个路由表表项（routing table entry）组成。静态路由所使用到的路由表项主要由"目的网络地址、网络掩码、下一跳地址、出接口"组成，目的网络地址和网络掩码用于确定目的网络的网络号，下一跳地址表示去往目标网络的下一跳 IP 地址，出接口表示数据包需要从本设备的哪个接口发送出去。



静态路由模拟软件可以定义为：一种用于模拟网络中静态路由配置和运行情况的计算机程序，旨在帮助网络工程师、学生等理解和实验静态路由的原理和工作方式。静态路由模拟实际上是网络仿真工具的一种功能，常见的网络仿真工具 GNS3（Graphical Network Simulator-3）、Cisco Packet Tracer、eNSP 等。

### 2. 目标

开发一款简单易用的静态路由模拟软件，具有以下功能：

- 允许用户手动配置主机的 IP 地址以及路由器的路由表
- 根据设置好的静态路由测试数据包的传输路径是否符合预期
- 能自动生成一张有代表性的网络拓扑图，包含路由器、交换机、部分主机站点

### 3. 功能需求

#### 3.1 路由器配置

- 用户能够使用图形界面配置路由器属性
- 用户能够添加和删除端口
- 能够将端口连接到其他设备上的端口
  用户选择线缆对象，点击需要连接的路由器/主机时会弹出端口列表下拉框，点击下拉框中的某个端口名，则能够将线缆连接到该端口。用户将线缆拖拽到另一路由器/主机，也会弹出下拉框供用户选择连接到哪个端口。
- 用户能够添加、删除、编辑路由表表项
  用户双击路由器显示路由表表项编辑界面
- 路由表条目包括目的网络地址、网络掩码、下一跳地址、出接口
- 路由器支持 8 个端口

#### 3.2 主机配置

- 用户能够使用图形界面配置主机
- 主机仅支持一个端口
- 主机能够配置 IP 地址、子网掩码、网关地址
- 用户能够添加和删除端口，将端口连接到路由器和交换机上

#### 3.3 交换机配置

- 用户能够使用图形界面配置交换机
- 交换机支持 24 个接口
#### 3.3 数据包模拟

- 用户能够输入源地址和目的地址来模拟数据包的传输。
- 软件根据路由表模拟数据包经过的路由路径，并展示给用户。

#### 3.4 生成拓扑图

- 能够生成有代表性的拓扑图示例

### 4. 非功能需求

#### 4.1 易用性

- 软件界面简洁明了，操作流程清晰。
#### 4.2 可扩展性

- 软件设计应考虑到未来可能的功能扩展，如动态展示路由转发的过程
- 为主机支持多个接口

#### 4.3 可靠性

- 软件应具有良好的稳定性和容错性，能够处理用户输入错误或异常情况。

### 5. 技术选型


采用 C++作为开发语言，Qt 作为图形界面的开发框架。C++具有丰富的面向对象特性，如封装、继承和多态，可以更好地组织和管理代码；C++是一种跨平台的编程语言，可以在不同操作系统上进行开发，并且具有很好的可移植性。

Qt 是一个跨平台的 C++ 应用程序开发框架，可以在多种操作系统上运行；Qt 提供了丰富的图形界面组件和配置项，开发者可以轻松创建各种用户界面，并实现丰富的显示效果；Qt 采用事件驱动的编程模型，使得程序的编写更加简洁和高效。
