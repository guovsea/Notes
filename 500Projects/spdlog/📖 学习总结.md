## 4.2 设计理念

- 核心思想

高可扩展性。能够派生 base_sink 以支持各种输出策略，比如 qt 对象、stdout、stderr 等。
能够派生 formatter 以自定义的格式格式化 log msg。

高可配置性。CMakeLists.txt 中支持许多编译选项，能够配置支持 pattern formatter，
例如：`[%H:%M:%S %z] [%n] [%^---%L---%$] [thread %t] %v`

高性能。支持异步日志


- 使用的设计模式

模板工厂模式：

`async_factory`

```cpp
// async logger factory - creates async loggers backed with thread pool.
// if a global thread pool doesn't already exist, create it with default queue
// size of 8192 items and single thread.
template <async_overflow_policy OverflowPolicy = async_overflow_policy::block>
struct async_factory_impl {
    template <typename Sink, typename... SinkArgs>
    static std::shared_ptr<async_logger> create(std::string logger_name, SinkArgs &&...args) {
        auto &registry_inst = details::registry::instance();

        // create global thread pool if not already exists..

        auto &mutex = registry_inst.tp_mutex();
        std::lock_guard<std::recursive_mutex> tp_lock(mutex);
        auto tp = registry_inst.get_tp();
        if (tp == nullptr) {
            tp = std::make_shared<details::thread_pool>(details::default_async_q_size, 1U);
            registry_inst.set_tp(tp);
        }

        auto sink = std::make_shared<Sink>(std::forward<SinkArgs>(args)...);
        auto new_logger = std::make_shared<async_logger>(std::move(logger_name), std::move(sink),
                                                         std::move(tp), OverflowPolicy);
        registry_inst.initialize_logger(new_logger);
        return new_logger;
    }
};

using async_factory = async_factory_impl<async_overflow_policy::block>;
using async_factory_nonblock = async_factory_impl<async_overflow_policy::overrun_oldest>;
```

模板方法模式

```cpp
SPDLOG_INLINE void logger::log_it_(const spdlog::details::log_msg &log_msg,
                                   bool log_enabled,
                                   bool traceback_enabled) {
    if (log_enabled) {
        sink_it_(log_msg);
    }
    if (traceback_enabled) {
        tracer_.push_back(log_msg);
    }
}
virtual void sink_it_(const details::log_msg &msg);
virtual void flush_();
```
`log_it_` 中使用的是虚函数： `sink_it_` and `flush_`, 子类可以重写这两个方法。


- 解耦策略

logger 层面的抽象策略

logger 用于提供繁多的接口方便用户使用，这些接口最终都依赖于少量的几个抽象接口。
`async_logger` 只需要重写 `sink_it_` 和 `flush_` 方法。logger 具体化了
info、set_formatter、backtrace、flush 接口，用于向下游提供稳定的接口，抽象
化了**将格式化后的日志内容的输出和刷新策略**。（最开始也没有这样的抽象，慢慢演进出来的）
