### 核心设计

#### 报文数据

图：数据类图

为了便于更好的抽象，本文在设计过程中使用多重继承的方式。本文将报文数据的实际内容封装在 NetwrokData 类中，表示最基本的数据类型。NetworkData 派生出的 NetworkFrame用于抽象数据链路层的以太网帧，它增加了两个数据成员：源 MAC 地址和目的 MAC 地址。

从 NetworkFrame 中派生出的 NetworkPacket 用于抽象网络层的数据包，它增加了两个数据成员：源 IP 地址和目的 IP 地址。为了实现 ARP 协议，本文从 NetworkPacket 派生出了 ArpData 用于抽象 ARP 报文，它增加了表示该 Arp 报文是请求报文还是响应报文。

每个数据类型都具有一个 type 函数，用于判断数据的动态类型。Type 函数的返回值为 DataType 类型的枚举变量，其取值为 DATA, FRAME, PACKET, ARP 中的某一个值。type 函数被声明我虚函数，其返回的类型由变量的动态类型决定。

当变量的动态类型为 NetwrokData，调用 type 函数会返回 DATA，当 NetworkFrame 调用 Type 函数时会返回 FRAME，依次类推。这样，只要接收 NetworkData 类型数据的函数，都能传入该继承体系中的所有类型的变量，并且能够知道变量的动态类型。

#### 节点设备

节点设备用于抽象静态路由所需要的网络设备，相关的类为： Device、DataLinkDevice、NetworkDevice、Switch、Host、Router。Device 用于抽象所有可以添加接口的网络设备，DataLinkDevice 用于抽象数据链路层的网络设备，NetworkDevice 用于抽象网络层的网络设备，Switch 用于抽象交换机，Host 和 Router 分别用于抽象主机和路由器。整个系统的类图如下：



##### 接口类的设计

Interface 类型的接口支持发送信息到对端的功能，DataLinkInteface 中包含全局唯一的 MAC 地址。NetworkInteface 中包含 IP 地址和子网掩码，用于确定该接口的 IP 地址和它所在的网络的网络号。

在添加接口时，根据 Device 的动态类型能够添加不同类型的接口。

##### 设备继承体系

Device 主要定义了连接接口的逻辑，接口同样也有抽象体系。 基类 Device 的功能是管理端口和向端口发送信息。Device 中有一个存储 Interface 指针，根据 C++动态绑定的特性，它能存储从 Interface 派生出的 DataLinkInteface 的指针和 NetworkInteface 的指针，从而很方便地对所有接口进行管理。Device 向端口发送信息时会调用端口的 Transfer 函数，这个过程并不会对数据做任何的修改。

派生自 Device 的 DataLinkDevice 类用于抽象数据链路层的设备，它并没有在 Device 添加新的功能。设计该层的意义是为了为以后添加新功能留出接口，比如要对以太网帧进行分片进行模拟。同时交换机的基于以太网帧的转发功能也不应该被网络层所继承，因此保留了 DataLinkDevice 作为中间层。

派生自 DataLinkDevice 的 NetworkDevice 用于抽象网络层的设备，它添加支持 ARP 协议的逻辑和路由表。将 ARP 协议放在该层的原因是主机和路由器都需要支持 ARP 协议，因为主机和路由器都工作在网络层，它们有时候需要根据 IP 地址获得该 IP 地址的设备的 MAC 地址。将路由表放 NetworkDevice 中的原因是为了模拟存在于主机和路由器中的路由表，路由器需要根据路由表确定路由转发的端口，主机在连接多个设备是同样需要通过路由表确定将数据包发送到哪个端口。

派生自 DataLinkDevice 的 Switch 类用于抽象交换机，负责转发以太网帧。

Host 和 Router 分别用于抽象主机和路由器，将在后续的内容中详细介绍。

##### Switch 类

Switch 类中存在一个 MacAddrTable，用于保存 MAC 地址和接口名的映射关系。

常见的交换机会根据收到数据帧中的源 MAC 地址建立该地址同交换机端口的映射，并将其写入 MAC 地址表中。交换机将数据帧中的目的 MAC 地址同已建立的 MAC 地址表进行比较，以决定由哪个端口进行转发。如数据帧中的目的 MAC 地址不在 MAC 地址表中，则向所有端口转发。

```
杨朔, 李长庚. 无线传感器网络泛洪时间同步协议安全算法[J]. 传感器与微系统, 2014, 33(1):137-140.
```


Switch 类的核心代码为：

```cpp
void Switch::Handle(NetworkFrame* frame, QString interfaceName)
{
	// 更新 MAC 表
	m_MacAddrTable[frame->sourceMacAddr] = interfaceName;
	// 处理逻辑
	if (frame->destMacAddr == BROADCAST_MAC_ADDRESS)
	{
		Broadcast(frame);
		return;
	}
	// 查询 MAC 表
	if (m_MacAddrTable.contains(frame->destMacAddr))
	{
		auto sendInterface = GetInterface(m_MacAddrTable[frame->destMacAddr]);
		Device::TransferToInterface(m_MacAddrTable[frame->destMacAddr], frame);
	}
		// MAC 地址不在 MAC 表中，则泛洪
	else
	{
		Broadcast(frame);
	}
}
```
当接收到来自某个的以太网帧时，它首先更新 MAC 地址和接口名映射表，如果该以太网帧的源 MAC 地址不在映射表中，则将该以太网帧的源 MAC 地址和该端口名的映射关系加入到映射表中然后向所有端口转发。如果该以太网帧的目的 MAC 地址在映射表中，则直接向该以太网帧的目的 MAC 地址所映射的端口转发。

##### NetworkDevice 类

该层主要用于处理 ARP 协议，包括处理接收的 ARP 数据和发送 ARP 请求。

**当 IP 数据报准备发送时，由数据链路层将它封装入以太网数据帧, 然后才能在以太网中传送。然而在封装过程中, 数据链路层并不知道以太网数据帧头中目的主机的 MAC 地址。唯-的信息是 IP 数据报头中的目的主机的 IP 地址。为了找到与目的主机 IP 地址相对应的 MAC 地址,根据地址解析协议 (ARP), 源主机会发送-个称作 ARP 请求的以太网数据帧给以太网上的每-个主机。这个过程称作 ARP 广播。**

```
[1]任侠,吕述望.ARP协议欺骗原理分析与抵御方法[J].计算机工程,2003(09):127-128+182.
```
ARP 协议的流程图如下：


```cpp
void NetworkDevice::HandleArpData(ArpData* arpData, NetworkInterface* interface)  
{  
   // 接收到 ARP 响应  
   if (arpData->flag == ArpData::RESPONSE)  
   {      assert(arpData->destMacAddr == interface->GetMacAddress());  
      assert(arpData->destIpAddr == interface->GetIpAddress());  
      // 更新 ArpTable      auto arpTable = GetArpTable();  
      arpTable[arpData->sourceIpAddr] = arpData->sourceMacAddr;  
      SetArpTable(arpTable);  
      emit ArpTableChanged();  
   }   // 响应 ARP 请求  
   if (arpData->flag == ArpData::REQUEST && arpData->destIpAddr == interface->GetIpAddress())  
   {      auto arpResponse = new ArpData;  
      arpResponse->flag = ArpData::RESPONSE;  
      arpResponse->sourceMacAddr = interface->GetMacAddress();  
      arpResponse->sourceIpAddr = interface->GetIpAddress();  
      arpResponse->destIpAddr = arpData->sourceIpAddr;  
      arpResponse->destMacAddr = arpData->sourceMacAddr;  
      Device::TransferToInterface(interface->GetName(), arpResponse);  
   }
}  
void NetworkDevice::BroadcastArpRequest(IpAddress destIpAddress) const  
{  
   for (auto interface : InterfaceList())  
   {      assert(interface->type() >= Interface::NWINTERFACE);  
      auto nwInterface = dynamic_cast<NetworkInterface*>(interface);  
      auto arpData = new ArpData;  
      arpData->flag = ArpData::REQUEST;  
      arpData->sourceIpAddr = nwInterface->GetIpAddress();  
      arpData->sourceMacAddr = nwInterface->GetMacAddress();  
      arpData->destIpAddr = destIpAddress;  
      arpData->destMacAddr = BROADCAST_MAC_ADDRESS;  
      TransferToInterface(nwInterface->GetName(), arpData);  
   }
}
```
HandleArpData 用于处理接收到的 ARP 数据包。当 ARP 数据包是响应（RESPONSE）时，函数会检查目标 MAC 地址和 IP 地址是否与当前网络接口相匹配，然后更新 ARP 表（ArpTable），将源 IP 地址和源 MAC 地址映射关系加入到 ARP 表中，并发出 ARP 表已更改的信号。

当 ARP 数据包是请求（REQUEST）时，并且目标 IP 地址与当前网络接口的 IP 地址相匹配时，函数会创建一个 ARP 响应数据包（arpResponse），然后通过设备的 TransferToInterface 函数将 ARP 响应数据包传输到网络接口。

`BroadcastArpRequest` 函数用于广播 ARP 请求数据包。它会遍历设备的所有网络接口，并为每个网络接口创建一个 ARP 请求数据包，将目标 IP 地址设置为参数提供的目标 IP 地址，并将目标 MAC 地址设置为广播地址（BROADCAST_MAC_ADDRESS），然后通过设备的 TransferToInterface 函数将 ARP 请求数据包传输到每个网络接口。

##### Router 类

路由器类主要负责转发数据包到不同的网段中.路由器根据数据包中的目标地址，通过查找路由表来确定下一跳的路径，并将数据包发送到正确的目标网络。

Router 类的核心代码为：

```cpp
void Router::Forward(NetworkPacket* packet)
{
	auto arpTable = GetArpTable();
	if (arpTable.contains(packet->destIpAddr)) {
		packet->destMacAddr= arpTable[packet->destIpAddr];
	} else {
		BroadcastArpRequest(packet->destIpAddr);
		QTimer::singleShot(100, this, [=](){ Forward(packet);});
		return;
	}
	auto routingTable = GetRoutingTable();
	for (auto entry : routingTable) {
		if ((packet->destIpAddr & entry.mask) == entry.dest) {
			packet->sourceMacAddr = NetworkDevice::GetInterface(entry.interfaceName)->GetMacAddress();
			TransferToInterface(entry.interfaceName, packet);
		}
	}
}`
```


Router 在接收到数据后会进行转发。转发的过程如下：首先检查目标 IP 地址是否已在 ARP 表中，若存在则将数据报的目标 MAC 地址设为目标 IP 所对应的 MAC 地址；如果目标 IP 地址是不存在于 ARP 表中，则进行广播 ARP 请求获取目标 MAC 地址，并设置定时器再次尝试转发发。在得到目标 MAC 地址之后，根据路由表规则匹配目标 IP 地址，设置数据包的源 MAC 地址为出接口的 MAC 地址，最后将数据包转发至相应出接口。

##### Host 类

主机主要功能有发送数据和处理数据。

主机的关键代码如下：
```cpp
bool Host::SendData(NetworkPacket* packet)  
{  
   auto arpTable = GetArpTable();  
   if (arpTable.contains(packet->destIpAddr)) {  
      packet->destMacAddr= arpTable[packet->destIpAddr];  
   } else  
   {  
      // Ip 地址在子网中  
      if ((packet->destIpAddr & m_SubnetMask) == (m_IpAddress & m_SubnetMask))  
      {         BroadcastArpRequest(packet->destIpAddr);  
      }      else  
      {  
         BroadcastArpRequest(m_Gateway);  
      }  
      QTimer::singleShot(100, this, [=](){ SendData(packet);});  
      return false;  
   }   auto interface = Device::InterfaceList()[0];  
   auto networkInterface = dynamic_cast<NetworkInterface*>(interface);  
   packet->sourceIpAddr = networkInterface->GetIpAddress();  
   // 修改源 MAC 地址  
   packet->sourceMacAddr = networkInterface->GetMacAddress();  
   Device::TransferToInterface(interface->GetName(), packet);  
   return true;  
}
void Host::Handle(NetworkPacket* packet, QString interfaceName)  
{  
   if (packet->type() == NetworkData::ARP)  
   {      auto arpData = dynamic_cast<ArpData*>(packet);  
      HandleArpData(arpData, GetInterface());  
      return;  
   }   emit OnReceived(packet);  
}
```

主机发送数据时，先获取主机的 ARP 缓存，并检查目标 IP 地址是否已经存在于 ARP 缓存中。如果目标 IP 地址已经存在于 ARP 缓存中，则直接设置数据包的目标 MAC 地址为 ARP 缓存中对应的 MAC 地址。否则，根据目标 IP 地址和主机的子网掩码进行比较，如果目标 IP 地址在子网中，则广播 ARP 请求以获取目标 MAC 地址；如果不在子网中，则广播 ARP 请求以获取网关（Gateway）的 MAC 地址。然后，设置定时器延迟100毫秒，重新调用 SendData 方法，并返回 false。最后，设置数据包的源 IP 地址和源 MAC 地址，并通过设备的 TransferToInterface 方法将数据包传输到网络接口。

主机接收到的数据包时，如果接收到的数据包是 ARP 类型的，则转换为 ARP 数据包并调用 HandleArpData 方法进行处理；否则，发出 OnReceived 信号通知界面部分该主机接收到数据包。
#### 数据传输过程

整个系统的执行逻辑主要依据 Qt 的信号/槽（signals/slots）机制。**Qt 中的信号/槽机制主要用于实现对象间的通讯。当一个特定事件发生的时候, 一个信号被发射, 并由一个被称作槽的函数来处理这一信号。信号/槽机制可以保证把一个信号和一个槽连接起来。** 

```
[1]黎逍,姜开永,李文达,等.基于Qt的轻量级商品配送系统设计[J].科学技术创新,2022(36):95-98.
```


信号/槽机制是对观察者模式的扩充。在 Qt 中使用 QObject:: connect 绑定一个信号和槽，其参数为：信号发出者，特定信号，信号接收者，接受者的槽函数。当绑定一个信号和槽时，信号接收者便开始观察信号发出者是否发送了特定的某个信号，一但信号发出者发出特定信号，信号接收者变回调用所绑定的槽函数。

图（**类图**）可以看出，Device 有一个 OnRecvData 的函数，这便是一个信号，当 Device 的某个端口接收到数据时该信号会被发出。DataLinkDevice、NetworkDevice 都有一个 OnRecv* 和 Received*函数。在 创建 DataLinkDevice 时绑定了底层 Device 的 OnRecvData 信号和 DataLinkDevice 的 RecviedData 的槽函数。因此当底层 Device 接收到数据（此时数据类型为 NetworkData*）时会发送 OnRecvData 信号，DataLinkDevice 的 RecviedData 槽函数会被调用。DataLinkDevice 的 RecviedData 函数会将 NetworkData*解析成 NetworkFrame 再向 NetworkDevice 层发送一个 OnRecvFrame 信号。NetworkDevice 接收到 OnRecvFrame 会向上层发送一个 OnRecvPacket 信号。

因此，在 DataLinkDevice 层之上的设备接收到的数据类型为 NetworkFrame*，在 NetworkDevice 层之上的设备接收到的数据类型为 NetworkPacket。

Switch 工作在 DataLinkDevice 层之上，当接收到 OnRecvFrame 信号时会调用 Handle 函数对接收到的 NetworkFrame 数据进行处理，也就是基于数据链路层的转发。Host 和 Router 工作在 NetworkDevice 层之上，当接收到 OnRecvPacket 信号是同样会对接收到的 NetworkPacket 数据进行处理。

### 界面设计

#### Qt 的 Graphics View 框架

在界面设计部分主要用到了 Qt 的 Graphics View 框架，下面介绍一些 Qt 的Graphics View 的一些关键特性和概念：

Qt 的 Graphics View 架构主要由以下三个部分组成：
 **场景（Scene）**：场景是Graphics View中的核心概念，它是一个2D的图形项容器，可以包含各种图形项，如线条、椭圆、多边形等。场景是一个逻辑上的概念，可以在其中添加、删除和操作图形项。
**图形项（Item）**：图形项是场景中的可视化对象，它可以是任何2D图形，如矩形、椭圆、文本、路径等。Qt提供了一些预定义的图形项类，同时也支持自定义图形项的创建。
**视图（View）**：视图是用于在屏幕上渲染场景的对象，它可以是QGraphicsView或其子类。视图提供了用户与场景交互的方式，例如平移、缩放、选择等操作。

此外 Qt 的 Graphics View 还具有很好的特性：
 **交互式操作（Interactive Operations）**：Graphics View 框架提供了丰富的交互式操作，包括平移、缩放、选择、拖放等，用户可以通过这些操作与场景中的图形项进行交互。利用它的交互式操作的特性，能够方便地实现支持拖拽的网络拓扑图。

 **事件处理**：Graphics View框架提供了丰富的事件处理机制，可以处理鼠标事件、键盘事件、焦点事件等，开发者可以通过重写事件处理函数来实现自定义的交互行为。

#####  MainWindow 类

`MainWindow` 类的主要功能是为用户提供一个可视化界面，用于创建和操作静态路由网络模拟器。通过这个界面，用户可以轻松地添加不同类型的网络设备，如路由器、PC 和交换机，并在它们之间绘制连接线，从而构建出一个网络拓扑图。

`MainWindow` 使用了 `QGraphicsView` 和 `DiagramScene` 自定义场景来管理和渲染拓扑图中的图形项。用户通过工具箱中的按钮来选择要添加的设备类型或连接线，并通过鼠标拖放的方式在视图中创建实例。每个设备和连接线都是一个 `DiagramItem` 对象，它们继承自 `QGraphicsItem`，并具有自己的属性和行为，可以响应用户的点击事件来弹出该设备的配置界面。当主机被点击后可以弹出主机配置界面，主机的配置界面由 HostDialog 生成，路由器的配置界面由 RouterDialog 生成。

此外，`MainWindow` 还提供了删除、选中项的功能，用户可以通过点击删除按钮来移除不再需要的设备或连接线。为了提高用户体验，`MainWindow` 还实现了移动模式，允许用户在场景中拖动设备，调整网络拓扑的结构。

#####  拓扑图相关类

DiagramItem、DiagramScene、Arrow 类主要用于实现拓扑图的类。

`DiagramItem`类是图形界面中的基本组件，代表网络拓扑图中的一个设备或连接点。每个`DiagramItem`都有一个关联的设备类型（Host、Switch、Router），并且可以与箭头（`Arrow`类）相连，表示设备之间的连接。`DiagramItem`设置了鼠标点击事件的处理逻辑，当某个 DiagramItem 被点击之后，会弹出相应类型的配置界面，用于配置它所关联的实际设备。

在创建和 Host 关联的 Diagramitem 对象时， Diagramitem 的槽函数 hostRecvdPackt 会连接到 Host 的 OnRecvPacket 信号。当 Host 接收到数据时，界面的 DiagramItem 闪烁一次。当 Host 被点击时，能够弹出测试界面，用户能够输入想要测试的 IP 地址，确认后 Host 会定时发送一次数据包。如果静态路由配置正确，该 IP 地址的 Diagramitem 应该会闪烁，否则静态路由配置错误。

`DiagramScene`类是管理这些`DiagramItem`的容器，它继承自`QGraphicsScene`，提供了一个场景，其中可以添加、移动和删除`DiagramItem`。`DiagramScene`有两个关键的成员变量：m_Mode 和 m_ItemType，用户点击左侧控制面板选择插入某个类型的节点后，m_ItemType 会被置为能表示该类型节点的枚举值并将 m_Mode 设置为插入模式；用户选择移动功能后，m_Model 会被设置为移动模式。

在插入模式下，DiagramScene 能够响应鼠标的点击事件和拖拽事件，在鼠标点击后根据 m_ItemType 的类型插入相应类型的节点，拖拽鼠标时会在鼠标的起点和终点之间绘制一条线，当鼠标的起点和终点都有 DiagramItem 时，则会将两个 DiagramItem 连接起来。在移动模式下，DiagramScene 能够响应鼠标的拖拽事件移动各个节点。

Arrow 类主要实现了观察者模式。Arrow 中存储着两个节点和一个 updatePositon 函数，当节点的位置发生改变时会调用 Arrow 对象的 updatePositon 函数通知 Arrow 对象，Arrow 将重新绘制连线。这样在移动节点的时候，连线也会同步更新。
