# 第二章 需求分析

## 2.1 目标

本研究旨在设计并开发一款简单易用的静态路由模拟软件，其主要目标为：

1. 允许用户手动配置主机的 IP 地址以及路由器的路由表。
2. 根据用户设置的静态路由信息，验证数据包的传输路径是否符合预期。
3. 能够自动生成一幅清晰的网络拓扑图，其中包含路由器、交换机以及部分主机站点。

## 2.2 功能需求

### 2.2.1 配置路由器属性

用户应能够通过直观的图形界面配置路由器的属性，主要包括以下功能：

4. 实现端口连接功能。
    - 用户选择线缆对象后，点击目标路由器或主机时应弹出端口列表下拉框。选择特定端口后，即可将线缆连接至该端口。用户也可通过拖拽线缆到其他路由器或主机，弹出下拉框进行目标端口选择。
5. 允许用户添加、删除、编辑路由表条目。
    - 双击路由器后应显示路由表的属性界面，用户可通过界面设置路由表。路由表条目包括目标网络地址、网络掩码、下一跳地址以及出接口信息。
6. 路由器需支持最多 8 个端口。

### 2.2.2 配置主机属性

用户应能够通过图形界面配置主机的属性并验证数据包传输路径是否符合预期，主要包括以下功能：

7. 实现端口连接功能。
8. 用户可在图形界面中配置 IP 地址、子网掩码以及网关地址。
9. 允许用户输入目的 IP 地址模拟数据包传输。
10. 用户可测试数据包传输路径是否符合预期。

## 2.3 非功能需求

系统的非功能性需求对用户体验和软件质量提升至关重要。设计静态路由实现与测试系统时，特别考虑以下非功能性需求：

11. **易用性**：软件界面应设计简洁清晰，操作流程易于理解。
12. **可扩展性**：考虑未来功能扩展可能性，如动态路由转发过程展示及主机支持多个接口等。
13. **可靠性**：软件应具备良好的稳定性和容错性，能够有效处理用户输入错误或异常情况。

## 2.4 系统模块设计

综合功能需求和非功能需求，系统可划分为界面和核心两大模块。具体功能模块划分如下：

- **界面模块**：包括拓扑图模块和菜单模块。拓扑图模块负责创建和管理节点，菜单模块生成内置拓扑图。管理模块具备连接、移动和删除节点等功能。
- **节点模块**：包含主机模块、路由器模块和交换机模块。主机模块用于配置主机属性和测试连接状态。
