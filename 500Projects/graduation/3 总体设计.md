### 界面设计

界面主要分为两个部分：左侧为控制台，包括以下组成元素：生成拓扑图的按钮、供用户选择将要生成的设备类型的选项、将鼠标拖拽动作用作移动设备位置或连接设备的选项。右侧为拓扑图界面，提供一个拓扑设计的画布，让用户可以拖拽节点（路由器、交换机、主机）来构建网络拓扑，支持连线操作，让用户可以连接各个设备。

用户能够右键某个节点弹出配置界面。不同类型的节点具有不同的配置界面，主机配置界面具有：主机名、网关地址、子网掩码、IP 地址四个输入框和一个确认按钮，用户点击确认按钮，主机的属性将被修改，用户关闭该界面主机的属性将不会被修改。

路由器配置界面主要分为三个部分：设备名称输入框、配置接口部分、配置路由表部分。配置接口部分有一个接口名称的下拉框，选择某个接口名称的下拉选项能够配置该接口的 IP 地址和子网掩码。配置路由表部分有网络号输入框、子网掩码输入框、下一跳地址输入框、添加按钮和路由表表项列表，在用户输入完成一个路由表表项后，点击添加按钮便能添加一条路由表表项并且出现在下方的路由表表项列表中。

### 核心设计

核心模块主要分为两个模块，报文数据和节点。报文数据用于抽象和模拟在各个节点中传输的数据，节点用于抽象网络设备中静态路由的部分。

报文数据主要有以太网帧（frame）和数据包（Packet），以太网帧是数据链路层协议数据单元，底层以太网（Ethernet）进行传输。**Ethernet 中传输的信号是以帧的形式存在的，是 Ethernet 通信信号的基本单元。在 Ethernet 中不允许非帧形式的信号存在，一旦发现有非帧形式的信号就会将其丢弃。** 

```
[1]王廷尧,马克城.以太网知识讲座(1)——以太网的基本概念[J].天津通信技术,2002(01):54-62.
```
数据包是网络层的数据单元，为了模拟数据包中除了有以太网帧的信息还有更多信息这一特点，可以从表示以太网帧的类型中派生出数据包类型，这样能传输以太网帧的数据也能传输数据包数据，同时也便于模拟封装和解析以太网帧的过程。

节点模块主要有三个子模块：主机、路由器和交换机。主机和路由器都是网络层的设备，交换机是数据链路层的设备。为了表示这一关系，可以让主机和路由器从用于抽象网络层设备的类中派生出来，交换机从用于抽象数据链路层的类中派生出来。

主机的主要功能有发送数据包和响应数据包。主机发送数据包时，主机会先填写数据包中的源 IP 地址、目标 IP 地址、源 MAC 地址、目标 MAC 地址，然后通过下层设备提供的发送数据的接口进行发送。当下层设备接收到数据包时会转发给主机，主机能够对数据包做出响应。

路由器的主要功能是路由转发。路由器在接收到数据是，能够根据用户手动配置的静态路由信息进行转发。

交换机的主要功能是交换以太网帧，当接收到以太网帧时，交换机能够学习并构建一张 MAC 地址到接口名的映射表，能够根据 MAC 地址进行转发。
