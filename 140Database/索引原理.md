## 索引的定义

数据之外，数据库系统还维护着**满足特定查找算法的数据结构**，这些数据结构以某种方式引用(指向)数据，这样就可以在这些数据结构上实现高级查找算法，这种数据结构就是索引。

索引的作用：

提高查询效率

- 索引的优势
  提高数据检索的效率，降低排序的成本
- 索引的劣势
  降低更新操作，如 INSERT、UPDATE、DELETE 的效率 (因为需要对索引结构进行修改)，占用空间，

## 索引数据结构

| 索引名称                                        | 描述                                                                             |
| ----------------------------------------------- | -------------------------------------------------------------------------------- |
| [[B+ 树#B+ 树\|B+ 树]]  [[B+ 树#B+ 树 \|B+ 树]] | B 树的变体，大部分引擎都支持 B+ 树索引                                           |
| Hash 索引                                       | 底层数据结构是用哈希表实现的, 只有精确匹配索引列的查询才有效, 不支持范围查询     |
| R-tree(空间索引)                                | 空间索引是 MyISAM 引擎的一个特殊索引类型，主要用于地理空间数据类型，通常使用较少 |
| Full-text(全文 索引)                            | 是一种通过建立倒排索引,快速匹配文档的方式。类似于 Lucene,Solr,ES                 |

引擎对索引的支持情况

| 索引                    | InnoDB | MyISM | Memory |
| ----------------------- | ------ | ----- | ------ |
| [[B+ 树#B+ 树 \|B+ 树]] | Y      | Y     | Y      |
| Hash 索引               | N      | N     | Y      |
| Full-text               | 5.6 后 | Y     | N      |
| R-tree                  | N      | Y     | N      |

MySQL 中优化之后的 [[B+ 树]]  (不重要，可视为还是原来的 B+树）

MySQL 索引数据结构对经典的 B+Tree 进行了优化。在原 B+Tree 的基础上，增加了反向的指针，提高区间访问的性能，利于排序。

## InnoDB 为什么选择使用 B+ 树

- 相较于二叉树，B+树层级更少，搜索效率高
- 相较于 [[B+ 树#B 树|B 树]] B+ 树的叶子节点仅作为索引节点，只需要保存索引值。这样一个页面能保存更多的子节点，降低树的高度，从而减少磁盘 IO 的次数。
- 相较于 Hash 索引，B+tree 支持范围匹配及排序操作

## MySQL 中的数据组织结构

### 聚集索引 & 二级索引

InnoDB 存储引擎中，根据索引的存储形式，又可以分为以下两种:

- 聚集索引 (Clustered Index)
  将数据存储与索引放到了一块，**索引结构的叶子节点保存了行数据**。必须有聚集索引, 而且只有一个
- 二级索引 (Secondary Index)
  将数据与索引分开存储，**索引结构的叶子节点关联的是对应的主键**

 二级索引中叶子结点存储的并不是数据的地址，而是聚集索引的索引值。因此，所有二级索引的都是先根据二级索引查到聚集索引的索引值，再根据聚集索引得到数据的地址，这种方式被称为 [[#回表查询]]。

### 回表查询

这种先到二级索引中查找数据，找到索引，然后再到聚集索引中根据索引，获取数据的方式，就称之为回表查询。

回表查询比较慢，因为要进行 2 次索引查询。
### 聚集索引产生规则

- 如果存在主键，主键索引就是聚集索引。
- 如果不存在主键，将使用第一个 UNIQUE NOT NULL 字段作为聚集索引。
- 如果表没有主键，且没有 UNIQUE NOT NULL 的字段，则 InnoDB 会自动生成一个 DB_ROW_ID 作为隐藏的聚集索引。
