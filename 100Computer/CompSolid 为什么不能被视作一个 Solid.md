# CompSolid 为什么不能被视作一个 Solid？

## 先说结论

> **在纯几何体积的意义上，CompSolid 确实可以合并成一个 Solid，而且很多 CAD 操作正是这样做的。但 OpenCascade 选择保留 CompSolid 这一层，是因为它承载了 Solid 丢失的关键信息。**

---

## 一、用反面论证：如果强行合并成 Solid 会怎样？

### 场景：两个不同材料的零件焊接在一起

```
钢块 + 铝块 焊接：

     ┌────────┬────────┐
     │  钢块  │  铝块  │
     │ Solid1 │ Solid2 │
     └────────┴────────┘
```

**作为 CompSolid 保留的信息：**
| 信息 | CompSolid | 合并成 Solid 后 |
|------|-----------|----------------|
| 有几种材料 | ✅ 两种（可分别标记） | ❌ 看不出 |
| 焊缝在哪里 | ✅ 共享 Face 就是焊缝 | ❌ Face 被删除了 |
| 各部分的体积 | ✅ 分别可查 | ❌ 只有一个总体积 |
| 各部分的质量 | ✅ 钢密度×V1 + 铝密度×V2 | ❌ 无法区分 |

> 合并后，中间那个 **共享 Face 被"消化"掉了**，两个区域变成一个均质体，所有区分信息 **不可逆地丢失**。

---

## 二、根本原因：共享面的归属问题

这是最深层的拓扑矛盾。

### 一个 Solid 的规则是：

> **外 Shell 的 Face 之间不能有"内部分隔面"。**

Solid 的外 Shell 必须是：
```
  封闭壳，且每个 Face 都是朝外的"皮肤"
```

### 但 CompSolid 中间有一张"内部隔板"：

```
     ┌────────┬────────┐
     │   A    │   B    │
     │        │        │
     └────────┴────────┘
              ↑
         这张共享 Face
         它既不是"外表面"
         也不是"内空腔表面"
         它是"A 和 B 之间的界面"
```

这张面有三个尴尬的属性：

| 属性 | 说明 |
|------|------|
| **它朝哪边？** | 它同时面向 A 和 B，不是单纯的外法线方向 |
| **它属于谁？** | 它同时属于 A 的 Shell 和 B 的 Shell |
| **它是什么性质？** | 它不是外表面，不是内腔面，它是 **界面** |

> Solid 的拓扑规则 **不允许** 这种"两面派"的 Face 存在。要么是外壳面，要么是内腔面，没有第三种身份。

---

## 三、数学定义上的冲突

### Solid 的数学定义（严格）

> 一个 Solid 是一个 **正则集合**，即：
> $$S = \text{closure}(\text{interior}(S))$$
> 
> 也就是说，Solid = 内部 + 边界，且边界是 **最小边界**。

如果 CompSolid 被视为一个 Solid，那中间的共享 Face 就成了：

```
一个在集合内部的"边界" → 矛盾！
边界必须在集合的外边缘
```

```
     ┌──────────────────┐
     │     内  部        │
     │  ┌──────────┐    │   ← 这张面在"内部"？
     │  │ 这算什么？│    │      如果在内部，它就不是边界
     │  └──────────┘    │      如果是边界，它就不该在内部
     │     内  部        │
     └──────────────────┘
```

> **数学上，正则集合的内部不能有边界。** 所以中间那张面在 Solid 的定义里没有合法身份。

---

## 四、实际操作中的策略

OpenCascade 并不是"不允许合并"，而是 **让用户自己选择**：

| 操作 | 结果 | 何时使用 |
|------|------|----------|
| **布尔 Fuse + Glue** | 保留 CompSolid | 需要保持材料/区域区分时 |
| **布尔 Fuse + Merge** | 合并为 Solid | 只关心总体积/外形时 |

```
用户的选择：

  需要保留内部界面 → 结果是 CompSolid
  不需要保留       → 结果是 Solid（内部面被移除）
```

---

## 五、回到根本：为什么不能直接取消这一层？

如果把 CompSolid 去掉，只用 Solid + Compound，会有三种情况：

| 情况 | 用什么表示 | 问题 |
|------|-----------|------|
| 单一实体 | Solid | ✅ 没问题 |
| 多个散落实体 | Compound | ✅ 没问题 |
| **多个粘合实体** | ❓ | ❌ Solid 不合适（丢失信息），Compound 不合适（丢失连接关系） |

第三种情况就是 CompSolid 存在的意义：

```
Solid    → "我是一个人"
Compound → "我们是路人"（无关系）
CompSolid→ "我们是连体婴"（有共享拓扑关系）
```

---

## 六、最终总结

**CompSolid 不能被视作一个 Solid 的三个根本原因：**

| # | 原因 | 本质 |
|---|------|------|
| 1 | **正则集合定义不允许内部有边界** | 数学矛盾 |
| 2 | **共享 Face 没有合法的拓扑身份** | 拓扑矛盾 |
| 3 | **合并会不可逆地丢失材料/区域/连接信息** | 工程信息丢失 |

> **Solid 是"一整块肉"，CompSolid 是"两块肉连着筋"。你当然可以把筋切断变成一整块，但那根筋（共享面）上承载的信息就永远丢失了。**
