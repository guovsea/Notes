# 递归中的入栈与出栈 —— 二叉树遍历专题

---

## 一、前置回顾

```
递归前 → 入栈序 → 先大后小（n → 1）
递归后 → 出栈序 → 先小后大（1 → n）
基线条件 → 栈底 → 转折点
```

二叉树的递归遍历，本质上就是把这「入栈 / 出栈」的规律，**从一条链扩展到了一棵树**。

---

## 二、二叉树递归的基本框架

```cpp
void traverse(TreeNode* node) {
    if (node == nullptr) return;       // 基线条件（空树）

    // ══════ 第一时机：入栈前 ══════
    // 此时刚进入 node，尚未访问任何子节点

    traverse(node->left);              // 递归左子树（入栈）

    // ══════ 第二时机：左子树出栈后 ══════
    // 左子树已全部处理完毕，回到 node

    traverse(node->right);             // 递归右子树（入栈）

    // ══════ 第三时机：右子树出栈后 ══════
    // 右子树已全部处理完毕，回到 node
}
```

每个节点会被**经过 3 次**，对应三种遍历：

| 遍历方式 | 处理时机 | 处理位置 | 顺序 |
|---------|---------|---------|------|
| **前序遍历** Pre-order | 第1次经过（入栈前） | 递归调用之前 | 根 → 左 → 右 |
| **中序遍历** In-order | 第2次经过（左子树出栈后） | 左递归后、右递归前 | 左 → 根 → 右 |
| **后序遍历** Post-order | 第3次经过（右子树出栈后） | 递归调用之后 | 左 → 右 → 根 |

## 三、三种遍历的完整代码

### 树结构定义

```cpp
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};
```

```cpp
/*
 *       1
 *      / \
 *     2   3
 *    / \

    TreeNode* root = new TreeNode(1);
    root->left  = new TreeNode(2);
    root->right = new TreeNode(3);
    root->left->left  = new TreeNode(4);
    root->left->right = new TreeNode(5);
    return root;
}
```

### 前序遍历（入栈前处理）

```cpp
void preorder(TreeNode* node) {
    if (node == nullptr) return;

    cout << node->val << " ";     // ① 入栈前：处理当前节点
    preorder(node->left);          // ② 递归左子树
    preorder(node->right);         // ③ 递归右子树
}

// 输出: 1 2 4 5 3
```

### 中序遍历（左子树出栈后处理）

```cpp
void inorder(TreeNode* node) {
    if (node == nullptr) return;

    inorder(node->left);           // ① 递归左子树
    cout << node->val << " ";     // ② 左子树出栈后：处理当前节点
    inorder(node->right);          // ③ 递归右子树
}

// 输出: 4 2 5 1 3
```

### 后序遍历（左右子树都出栈后处理）

```cpp
void postorder(TreeNode* node) {
    if (node == nullptr) return;

    postorder(node->left);         // ① 递归左子树
    postorder(node->right);        // ② 递归右子树
    cout << node->val << " ";     // ③ 全部出栈后：处理当前节点
}

// 输出: 4 5 2 3 1
```

## 五、用「入栈 / 出栈」视角理解遍历顺序

### 访问时间轴

把每个节点看作一个递归函数调用，**每个节点有 3 个时间点**可以被处理：

```cpp
void traverse(TreeNode* node) {
    if (!node) return;

    // ──── 时间点 A：刚进入（入栈前）────────
    //   前序遍历在这里处理 → 先遇到谁就先处理谁
    //   最先遇到的是根 → 所以 根 → 左 → 右

    traverse(node->left);

    // ──── 时间点 B：左子树返回后（左出栈，右未入栈）────
    //   中序遍历在这里处理 → 左子树全部处理完，轮到根
    //   所以 左 → 根 → 右

    traverse(node->right);

    // ──── 时间点 C：右子树返回后（全部出栈）────────
    //   后序遍历在这里处理 → 子树全部处理完才处理根
    //   所以 左 → 右 → 根
}
```

### 直觉理解

| 遍历 | 直觉 | 类比 |
|------|------|------|
| **前序** | 一见到就处理 | 像老板巡视，进门就下指令 |
| **中序** | 左手边忙完再处理 | 像读完左页再翻右页 |
| **后序** | 全部忙完才处理 | 像做完总结报告，先干活后汇报 |

## 六、进阶应用

### 6.1 层序遍历与递归的关系

层序遍历（BFS）不属于递归的入栈/出栈模式，而是用**队列**实现的。但可以用递归模拟：

```cpp
// 用递归 + 深度标记实现层序输出
void levelOrder(TreeNode* node, int level, vector<vector<int>>& result) {
    if (!node) return;
    if (result.size() == level) result.push_back({});
    result[level].push_back(node->val);        // 入栈前处理（类似前序）
    levelOrder(node->left,  level + 1, result);
    levelOrder(node->right, level + 1, result);
}
// result: [[1], [2,3], [4,5]]
```

### 6.2 翻转二叉树（前序 vs 后序都可以）

```cpp
// 前序风格：入栈前交换
TreeNode* invertTree_pre(TreeNode* root) {
    if (!root) return nullptr;

    swap(root->left, root->right);     // 入栈前交换
    invertTree_pre(root->left);
    invertTree_pre(root->right);
    return root;
}

// 后序风格：出栈后交换
TreeNode* invertTree_post(TreeNode* root) {
    if (!root) return nullptr;

    invertTree_post(root->left);
    invertTree_post(root->right);
    swap(root->left, root->right);     // 出栈后交换
    return root;
}
```

> 💡 两种结果相同！因为交换左右子树是**对当前节点的操作**，只要每个节点都被处理一次即可。

### 6.3 求树的最大深度（后序的典型应用）

```cpp
// 后序思想：先知道子树深度，才能算出当前深度
int maxDepth(TreeNode* root) {
    if (!root) return 0;

    int leftDepth  = maxDepth(root->left);    // 左子树出栈后得到结果
    int rightDepth = maxDepth(root->right);   // 右子树出栈后得到结果

    return max(leftDepth, rightDepth) + 1;    // 全部出栈后汇总
}
```

### 6.4 判断 BST 合法性（中序的典型应用）

```cpp
// 中序思想：BST 的中序遍历一定是严格递增的
bool isValidBST(TreeNode* root) {
    TreeNode* prev = nullptr;
    return inorderCheck(root, prev);
}

bool inorderCheck(TreeNode* node, TreeNode*& prev) {
    if (!node) return true;

    // 左子树
    if (!inorderCheck(node->left, prev)) return false;

    // 中序位置：检查是否严格递增
    if (prev && prev->val >= node->val) return false;
    prev = node;

    // 右子树
    return inorderCheck(node->right, prev);
}
```

### 6.5 最近公共祖先（后序的典型应用）

```cpp
// 后序思想：只有子树都查完了，才能判断当前节点是不是 LCA
TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
    if (!root || root == p || root == q) return root;

    TreeNode* left  = lowestCommonAncestor(root->left,  p, q);  // 左子树出栈
    TreeNode* right = lowestCommonAncestor(root->right, p, q);  // 右子树出栈

    // 后序位置：汇总左右结果
    if (left && right) return root;        // p、q 分布在两侧 → 当前就是 LCA
    return left ? left : right;            // 都在某一侧 → 传递上去
}
```

### 6.6 路径总和（前序的典型应用）

```cpp
// 前序思想：一进入节点就累加，到达叶子立刻判断
bool hasPathSum(TreeNode* root, int targetSum) {
    if (!root) return false;

    targetSum -= root->val;                        // 入栈前：立即累减

    if (!root->left && !root->right)               // 叶子节点：直接判断
        return targetSum == 0;

    return hasPathSum(root->left, targetSum) ||
           hasPathSum(root->right, targetSum);
}
```

## 6.4 判断 BST 合法性（中序）

### 先搞懂 BST 是什么

BST（二叉搜索树）有一条铁律：**左 < 根 < 右**，对整棵树的每个节点都成立。

```
合法 BST:
        5
       / \
      3   7
     / \   \
    2   4   8

非法 BST:
        5
       / \
      3   7
     / \   \
    2   6   8      ← 6 在 5 的左子树里，但 6 > 5，违反规则！
```

### 核心思路

中序遍历 BST，输出一定是**严格递增**的。所以我们只需要检查：**中序遍历中，每个节点是否比前一个大**。

```
合法 BST 的中序: 2 → 3 → 4 → 5 → 7 → 8    ✅ 严格递增
非法 BST 的中序: 2 → 3 → 6 → 5 → 7 → 8    ❌ 6 > 5 不递增！
```

### 为什么要用中序？前序/后序不行吗？

```
非法 BST:
        5
       / \
      3   7
     / \   \
    2   6   8

前序遍历: 5 3 2 6 7 8  → 你无法一眼看出 6 不该在 5 的左子树下
后序遍历: 2 6 3 8 7 5  → 同样混乱
中序遍历: 2 3 6 5 7 8  → 一看就知道 6→5 逆序了！❌
```

> 💡 中序遍历的顺序是「左 → 根 → 右」，恰好就是**从小到大**的顺序，所以中序天然适合检查 BST。

### 完整图解

```
检查这棵树是否是合法 BST:

        5
       / \
      3   7
     / \   \
    2   4   8


中序遍历过程（prev 始终指向上一个访问的节点）:

    步骤1: 到达节点2（最左）
           prev=null     → 2>null ✅（第一个节点，跳过比较）
           prev=2

    步骤2: 回到节点3
           prev=2        → 3>2 ✅
           prev=3

    步骤3: 到达节点4
           prev=3        → 4>3 ✅
           prev=4

    步骤4: 回到节点5
           prev=4        → 5>4 ✅
           prev=5

    步骤5: 到达节点7
           prev=5        → 7>5 ✅
           prev=7

    步骤6: 到达节点8
           prev=7        → 8>7 ✅
           prev=8

    全部通过 → 是合法 BST ✅
```

### 代码逐行注释

```cpp
class Solution {
    TreeNode* prev = nullptr;  // 全局记录"上一个访问的节点"

public:
    bool isValidBST(TreeNode* root) {
        if (root == nullptr) return true;

        // ──── 1. 先遍历左子树 ────
        if (!isValidBST(root->left)) return false;

        // ──── 2. 中序位置：检查当前节点 ────
        // prev 是中序遍历中排在 root 前面的那个节点
        // BST 要求严格递增，所以 prev->val 必须 < root->val
        if (prev != nullptr && prev->val >= root->val)
            return false;

        prev = root;  // 更新 prev，让下一个节点来和当前比较

        // ──── 3. 再遍历右子树 ────
        if (!isValidBST(root->right)) return false;

        return true;
    }
};
```

## 6.5 最近公共祖先 LCA（后序）

### 什么是 LCA？

```
        3
       / \
      5   1
     / \ / \
    6  2 0  8
      / \
     7   4

问：节点 7 和 4 的最近公共祖先是谁？

从 7 往上走: 7 → 2 → 5 → 3
从 4 往上走: 4 → 2 → 5 → 3
               ↑
              2 是第一个碰面的祖先 → LCA(7,4) = 2
```

### 为什么必须用后序？

> **你必须先知道子树里有没有 p 或 q，才能判断当前节点是不是 LCA。**
> 这就意味着：左子树的结果、右子树的结果，都得先算出来 → **后序遍历**。

### 递归的返回值含义

```
返回值 = 在以 root 为根的子树中，找到了谁？

- 返回 nullptr  → 这棵子树里既没有 p 也没有 q
- 返回 p        → 这棵子树里找到了 p（q 没找到）
- 返回 q        → 这棵子树里找到了 q（p 没找到）
- 返回某个节点  → 这个节点就是 LCA（p 和 q 分布在它的两侧）
```

### 完整图解：LCA(7, 4)

```
        3
       / \
      5   1
     / \ / \
    6  2 0  8
      / \
     7   4

后序遍历：先处理左右子树，最后处理自己（自底向上）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

节点6:  左=nullptr, 右=nullptr
        不是p也不是q → return nullptr

节点7:  左=nullptr, 右=nullptr
        7 == p → return 7          ← 找到 p！

节点4:  左=nullptr, 右=nullptr
        4 == q → return 4          ← 找到 q！

节点2:  左=7, 右=4
        ┌─────────────────────────────────┐
        │ left=7  且  right=4            │
        │ 说明 p 和 q 分布在 2 的两侧！   │
        │ → 2 就是 LCA！return 2         │
        └─────────────────────────────────┘

节点5:  左=nullptr, 右=2
        left=nullptr，right=2
        → 只有右边找到了东西，把 2 传上去
        → return 2

节点0:  左=nullptr, 右=nullptr → return nullptr
节点8:  左=nullptr, 右=nullptr → return nullptr

节点1:  左=nullptr, 右=nullptr → return nullptr

节点3:  左=2, 右=nullptr
        left=2，right=nullptr
        → 只有左边找到了东西，把 2 传上去
        → return 2

最终答案: 2 ✅
```

### 另一种情况：LCA(5, 4)

```
        3
       / \
      5   1
     / \
    6  2
      / \
     7   4

p=5, q=4

后序自底向上:

节点6: return nullptr
节点7: return nullptr
节点4: return 4（4==q）

节点2: 左=nullptr, 右=4
       → 一边有结果，传递上去 → return 4

节点5: 左=nullptr, 右=4
       但！5 == p
       ┌─────────────────────────────────────┐
       │ 自己就是 p！而且子树里找到了 q(=4)  │
       │ → 5 自己就是 LCA！return 5          │
       └─────────────────────────────────────┘

最终答案: 5 ✅

解释：5 是 4 的祖先，同时 5 自己就是 p，
      所以 LCA 就是 5 自己
```

### 代码逐行注释

```cpp
TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
    // 基线条件：走到底了，或者找到了 p/q
    if (root == nullptr || root == p || root == q)
        return root;

    // ──── 后序：先问左右子树 ────
    TreeNode* left  = lowestCommonAncestor(root->left,  p, q);
    TreeNode* right = lowestCommonAncestor(root->right, p, q);

    // ──── 后序位置：汇总左右结果 ────

    // 情况1: 左右都有结果 → p 和 q 分布在 root 两侧
    if (left != nullptr && right != nullptr)
        return root;   // root 就是 LCA！

    // 情况2: 只有左边有结果 → p 和 q 都在左子树
    // 情况3: 只有右边有结果 → p 和 q 都在右子树
    // 两种情况都是"把非空的那一侧传上去"
    return left != nullptr ? left : right;
}
```

```
LCA(7, 4) 的返回值自底向上传递:

        2          ← 最终返回给调用者
       / \

   6→null  7→7  4→4

```

## 6.6 路径总和（前序）

### 什么是路径总和？

```
        5
       / \
      4   8
     /   / \
    11  13  4
   / \       \
  7   2       1

targetSum = 22

问：是否存在一条「从根到叶子」的路径，节点值之和 = 22？

路径1: 5 → 4 → 11 → 7  = 27  ❌
路径2: 5 → 4 → 11 → 2  = 22  ✅ 找到了！
路径3: 5 → 8 → 13      = 26  ❌
路径4: 5 → 8 → 4 → 1   = 18  ❌
```

### 为什么用前序？

> **一进入节点就累减 targetSum**，到达叶子时看是否恰好减到 0。
> 不需要等子树返回什么结果，只需要**一边走一边减** → 前序。

### 完整图解

```
        5           targetSum = 22
       / \
      4   8
     /   / \
    11  13  4
   / \       \
  7   2       1


前序遍历（入栈时做决策）:

━━━ 进入节点5 ━━━
remaining = 22 - 5 = 17

  ━━━ 进入节点4 ━━━
  remaining = 17 - 4 = 13

    ━━━ 进入节点11 ━━━
    remaining = 13 - 11 = 2

      ━━━ 进入节点7 ━━━
      remaining = 2 - 7 = -5
      这是叶子节点！remaining=-5 ≠ 0 ❌

      ━━━ 进入节点2 ━━━
      remaining = 2 - 2 = 0
      这是叶子节点！remaining=0 ✅ 找到了！return true

最终: true ✅
```

### 代码逐行注释

```cpp
bool hasPathSum(TreeNode* root, int targetSum) {
    // 基线条件：空节点，没找到
    if (root == nullptr) return false;

    // ──── 前序位置：一进来就扣减 ────
    targetSum -= root->val;

    // 叶子节点：左右都为空 → 这是一条完整路径的终点
    if (root->left == nullptr && root->right == nullptr)
        return targetSum == 0;  // 恰好减完 = 找到了

    // 不是叶子 → 继续往左右子树找
    // 只要任一边找到就行（用 || 连接）
    return hasPathSum(root->left, targetSum)
        || hasPathSum(root->right, targetSum);
}
```

```

│
├── hasPathSum(4, 17)          ← 22-5=17
│   │

│       │

│       │
│       └── hasPathSum(2, 2)     ← 13-11=2
│           └── 2 是叶子: return (2-2 == 0) = true ✅
│
```

## 终极对比总结

```
┌──────────┬──────────┬──────────────────────────────────────────┐
│          │  前序     │  一进来就处理，不等子树                    │
│          │  (入栈前) │  "我能给子节点带什么下去？"               │
│          │          │  例：路径总和 → 带 remaining 往下减       │
├──────────┼──────────┼──────────────────────────────────────────┤
│  一棵树  │  中序     │  左子树处理完，轮到中间，再去右子树        │
│  三种     │  (左出栈后)│ "从左到右的有序性"                      │
│  时机     │          │  例：BST 验证 → 检查递增序列              │
├──────────┼──────────┼──────────────────────────────────────────┤
│          │  后序     │  子树全部处理完，最后汇总                  │
│          │  (全出栈后)│ "子节点能给我返回什么？"                  │
│          │          │  例：LCA → 先问左右，再判断自己            │
└──────────┴──────────┴──────────────────────────────────────────┘

选哪种？问自己一个问题:
  - 需要先处理自己再传递给子树？    → 前序
  - 需要利用"左到右"的顺序？        → 中序
  - 需要子树的结果才能做判断？       → 后序
```
