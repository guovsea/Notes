
## 指定存储引擎
```sql
CREATE TABLE 表名(  
字段1 字段1类型 [ COMMENT 字段1注释 ] , 
...  
字段n 字段n类型 [COMMENT 字段n注释 ]
) ENGINE = InnoDB [ COMMENT 表注释 ] ;

show engines ; -- 查看引擎支持情况
```


## 存储引擎

### InnoDB

**特点**

- DML 操作遵循 ACID 模型，支持事务
- 行级锁，提高并发访问性能
- 支持外键 FOREIGN KEY 

#### [[InnoDB架构]]

InnoDB 的内存结构和磁盘结构
#### 逻辑存储结构

![[InnoDB逻辑存储结构.png]]

**表空间** -> 段 -> 区 -> 页 -> 行

**段 (Segment)**: 表空间是由各个段组成的，常见的段有数据段、索引段、回滚段等。InnoDB 中对于段的管理，都是引擎自身完成，不需要人为对其控制，一个段中包含多个区。

**区 (Extent)**: 区是表空间的单元结构，每个区的大小为1M。一个区中一共有64个连续的页。
  
**页 (Page)**: 页是组成区的最小单元，页也是 InnoDB 存储引擎磁盘管理的最小单元。每个页的大小默认为 16KB。为了保证页的连续性，InnoDB 存储引擎每次从磁盘申请 4-5 个区。

**行 (Rwo)**: InnoDB 存储引擎是面向行的，也就是说数据是按行进行存放的，在每一行中除了定义表时所指定的字段以外，还包含两或三个个隐藏字段

#### 隐藏字段

- `DB_TRX_ID`：最后对该行进行插入或修改的事务 ID
- `DB_ROLL_PTR`：回滚指针，指向该数据的上一个版本，用于维护历史版本链 [[undo log]]
- `*DB_ROW_ID`：隐藏的聚集索引，可能会有的字段，没有可作为聚集索引的字段则会生成。InnoDB 中必然会有一个主键。[[索引使用#聚集索引产生规则|聚集索引产生规则]]


### MyISM

MyISAM是MySQL早期的默认存储引擎。

**特点**

- 不支持事务
- 不支持外键
- 只支持表锁，不支持行锁
- 访问速度快

### Memory

**特点**

- 支持 hash 索引

### 区别

| 特点         | InnoDB        | MySIM | Memory |
| ------------ | ------------- | ----- | ------ |
| 存储限制     | 64TB          | 256TB | RAM    |
| 事务         | 支持          | -     | -      |
| 锁机制       | 行锁          | 表锁  | 表锁   |
| B+tree 索引  | 支持          | 支持  | 支持   |
| hash 索引    | -             | -     | 支持   |
| 全文索引     | 支持 (5.6 后) | 支持  | -      |
| 空间使用     | 高            | 低    | N/A    |
| 内存使用     | 高            | 低    | 中等   |
| 批量插入数据 | 低            | 高    | 高     |
| 支持外键     | 支持          | -     | -       |
