## 模板参数列表

1. 类型参数

每个类型参数前都应有 `typename` 关键字。

定义时，`typename T` 等价于 `int a`；使用时，`typename T` 等价于

```cpp
template <typename T, U &u> // 如果不加 typename，编译器在解析 U 时就不知道是模板的类型参数还是真正的类类型
T foo(const T &t) {
}
template <typename T, typename U> 
T foo(const T &, const U &) {
}
```

1. 非类型参数

**非类型参数具有常量属性**

非类型参数，可以是一个整型，或者是一个**指向对象或函数类型的指针或引用**。

```cpp
template <typename T, unsigned M>
T foo(const T&) {
    new T[M];
}
```

非类型参数的实参，绑定到整型参数时必须是一个常量表达式。绑定到指针或引用时**必须有静态的生存周期**

```cpp
class U {
  public:
    void hello(void) { std::cout << "hello U" << std::endl; }
};

template <typename T, U &u> T foo1(const T &t) {
    std::cout << t << std::endl;
    u.hello();
}

U u;    // 正确

int main(int argc, char const *argv[]) {
    U u; // 编译错误：没有与参数列表匹配的
    foo1<int, u>(3);
    return 0;
}

```

在模版定义内，模板非类型参数是一个**常量**。在需要常量表达式的地方（比如数组大小）可以使用模板非类型参数

### 类模板

**类模板的模板名并不是类型名，`模版名<参数列表>` 才是类型名**

在类模板作用域中，可以将 `模版名<参数列表>` 这样的类型名简写成 `模板名`

```cpp
template <typename T>
class Num{
    public:
    typedef T value_type;
    T get();
    // value_type get();  // 也可以使用类型别名
    Num& operator++() { // 前置++
    // Num<T>& operator++() { // 前置++ 当然也可以使用 模板名<参数列表>作为类型名
        val_++;
        return *this;
    }
    Num operator++(int);
    private:
    T val_;
};

template<typename T>
// 作用域外使用模板定义的类型名时，必须提供模板参数列表
Num<T> Num<T>::operator++(int) {
    // 作用域内可以直接使用模板名，不提供模板参数列表
    Num res = *this;
    ++*this;
    return res;
}

template<typename T>
// typename 是必须的，否则编译器无法确定是类型名还是static变量（static变量可以通过类名::访问）
typename Num<T>::value_type Num<T>::get() {
    return val_;
}

```
默认情况下，C++假定通过作用域运算符访问的名字不是类型。因此在使用模板类的类型成员时，必须使用 `typename` 显式告诉编译器该名字是一个类型。

在类外定义函数时，如果返回值的类型要使用类中定义的类型名时，要在类名前使用 `typename` 限定。

## 模板和友元

1. **具体类**和模板以该类为实参的实例
   需要前置声明，因为使用的是类型名，**模板的前置声明：`class 模板名<参数列表>` 就是用来声明一个类型名的**。
2. **具体类**和模板的所有实例
   不需要前置声明，使用的是模板的定义
3. **模板**实例和另一个模板参数相同的实例
   需要前置声明，使用的类型名
4. **模板**的每个实例和另一个模板的每个实例
   不需要前置声明，使用的是模板的定义
5. **模板**的每个实例和具体类
   不需要前置声明，具体类就是类型名
6. **模板**的每个实例和自己的类型参数为友元
```cpp
template <typename> class F;  // 前置声明，将模板的一个特定实例声明为友元时需要
class C {
    // 具体类和模板以该类为实参的实例
    friend class F<C>;  // 需要前置声明
    // 具体类和模板的所有实例
    template <typename T> friend class F2;
};

template <typename T> class C2 {
    // 模板实例和另一个模板参数相同的实例
    friend class F<T>; // 需要前置声明
    // 模板的每个实例和另一个模板的每个实例
    // 必须使用与类模板本身不通的模板参数
    template <typename X> friend class F2; // 不需要前置声明
    // 模板的每个实例和具体类
    friend class C1; // 不需要前置声明
    // 模板的每个实例和自己的类型参数为友元
    friend T;
};
```

## 模板类型别名

**模板不是类型**（模板：`Num<T>`；模板名：`Num`）
```cpp
// 模板不是类型
// typedef Num<T> // 错误
typedef Num<in> IntNum;  // 正确，模板<实参列表> 是类型
// 新标准允许使用 using 定义模板类型别名（但并不是类型别名）
template <typename T> using twin = pair<T,T>;
twin<string> name; // 等价于 pair<string, string> name;

// 定义模板类型别名时可以固定模板实参
template <typename T> using partNo = part<T, unsigned>;
partNo<string> books; // 等价于 pair<string, unsigned> books;
```

## std::less
```cpp
template<typename T, typename F = less<T>>
int compare(const T &a, const T &b, F f = F()) {
    if (f(a, b)) return -1;
    if (f(b, a)) return 1;
    return 0;
}
```
less 是一个 struct 类，它只有一个成员 `operator()`。在使用 `compare` 的时候，如果不指定类型参数 F，那么 F 将是 `std::less<T>` 类型；`F f` 则限定 `f` 应该是一个 `std::less<T>` 对象。`F()` 等价于 `std::less<T>()`，**`F()` 会使用默认构造函数创建一个 `F` 类型 (也就是 `std::less<T>` 类型）的对象**。*`F()` 并不是一个函数*

## 空模板实参列表


如果模版类为所有模板参数都提供了默认实参，且我们希望使用这些默认实参，也必须在模板名后跟上 `<>`。因为**模板名并不是类型名，`模板<实参列表>` 才是类型名，即使实参列表为空**

```cpp
template <class T = int> class Numbers {

};
int main(int argc, char const *argv[]) {
    // Numbers n; // 报错，需要实参列表
    Numbers<> n;
}
```

## 显式实例化

当模板被使用时才会进行实例化，这一特性意味着相同的实例可能出现在多个文件中。当两个或多个独立编译的源文件使用了相同的模板并提供相同的模板参数，每个文件中都会有一个该模板的实例。

```cpp
extern template 类或函数声明;  // 实例化声明
template 类或函数声明;         // 实例化定义
```
可以在使用该模板前，先进行声明，这样本文件中就不会生成实例化代码

```cpp
// extern 模板声明，编译器不会在本文件中生成实例化代码
extern template class Bolb<string>;
extern template int compare(const int&, const int&);
```
但是必须在程序的其他位置有一个该实例化的定义。*也就是不加 `extern` 关键字*

```cpp
// 某处必须有定义，编译器将会在该文件中生成实例化代码
template class Bolb<strng>;
template int compare(const int&, cosnt int&);
```

**对每个实例化声明，在程序中某个位置必须有其显式的实例化定义**。否则，将会在链接阶段产生链接错误。

**实例化定义会实例化类的所有成员**。**用来实例化一个类模板的类型必须能够用于模板的所有成员。**

实例化定义必须在 `.cpp` 文件中，而不是在 `.h` 文件中

```cpp
/// template.h
#include <iostream>
template<typename T>
void print_(const T & t) {
    std::cout << t << std::endl;
}
// template void print_(const double &);  // note: explicit instantiation definition is here

/// main.cpp
#include "template.h"
extern template void print_(const double &);
int main(void) {
    print_(3.14);
    return 0;
}

/// temp.cpp
#include "template.h"
template void print_(const double &);
```
## 函数模板显式实参

**显式模板实参推断按从左往右的顺序与对应的模板参数列表进行匹配**。

**应该把需要显式指定的模板实参放在参数列表的最前面，后面未指定的模板参数应该能够从函数参数推断出来。**

```cpp
// T1 需要指定实参，T2 T3 都可以推断出来
template<typename T1, typename T2, typename T3>
T1 add(const T2 &a, const T3 &b) {
    return a + b;
}
// 糟糕的设计，需要指定实参的 T3 在模板参数列表的最后一个，因此要给参数列表中的所有参数指定实参
// template<typename T1, typename T2, typename T3>
// T3 add(const T1 &a, const T2 &b) {
//     return a + b;
// }
int main(void) {
    std::cout << add<int>(3.8, 4.5) << std::endl;
    // std::cout << add<double, double, int>(3.8, 4.5) << std::endl;
}
```
**模板类型参数已经显式指定了的函数实参也能进行正常的类型转换**

```cpp
long lng;
compare(lng, 1024); // 错误，模板参数不匹配
compare<long>(lng, 1024); // 正确，实例化 compare(long, long)
// 1024 本来为 int 类型，正常地转换成了 long 类型
```

## 尾置返回值

**利用函数模板中的实参类型推导出返回值类型时需要使用尾置返回值**。

```cpp
template <typename It>
// 尾置返回值必须在函数名前面用 auto 占位
auto func(It beg, It end) -> decltype(*beg) {
//
}

// 原因，如果不使用尾置返回值，当编译器处理到 ??? ，虽然可以用`decltype(*beg)`来
// 确定返回值的类型，但是在处理完函数的参数列表之后，才知道 beg 的存在。因此要在参数
// 列表之后通过函数的参数 decltype 出返回值类型的引用
template <typename It>
??? func(It beg, It end) {
//
}
```
[[decltype]]

decltype 对某个左值（`*It` 返回的就是左值）使用时，**推断出的类型是该左值类型的引用**。

*只用 decltype 的话那每次的返回值类型都只能是引用，那肯定是不 OK 的。这样就不能返回局部对象了，因为不能返回局部对象的引用*

## 类型转换的标准库模板类

头文件: **type_traits**

脱去 decltype 的引用，获得元素本身的类型：

```cpp
remove_reference<decltype(*beg)>::type

// 返回类型为 *It 的类型，而不是其引用
template <typename It>
auto func(It beg, It end) -> 
    typename remove_reverence<decltype(*beg)>::type {
//
return *beg;  // 返回 *beg 的拷贝
}
```

## 模板实参推断和函数指针

当用一 个函数模板初始化一个函数指针，或为一个函数指针赋值时，编译器使用函数指针的类型来推断模板实参。

```cpp
template<typename T>
void print(const T& a) {
    std::cout << a << "  " << "sizeof a = " << sizeof(a) << std::endl;
}
int main(void) {
    void (*p1)(const int&);
    p1 = print;
    p1(3);
    void (*p2)(const double &);
    p2 = print;
    p2(3.24);
}
```
output:
```text
3  sizeof a = 4
3.24  sizeof a = 8
```
如果推断有歧义，可以使用 [[#显式实例化]]消除歧义，实例化一个具体的函数

## 模板实参推断和引用类型的函数形参

**模板类型参数已经显式指定了的函数实参也\能进行正常的类型转换**

函数形参为非引用

- `template <typename T> void f(T t);` 
- `template <typename T> void f(const T t);`

推断出的 T 的类型都只会是普通类型 X，const 只用来修饰函数形参 t。

函数形参为引用

`template <typename T> void f(T &p);` 

- 编译器会应用正常的绑定规则
- const 是底层的，不是顶层的

| 实参类型/形参类型 | const T&，万能绑定 | T&，只有左值能绑定 |
| --------- | ------------- | ---------- |
| const X   | X             | const X    |
| X         | X             | X          |
| 右值        | 可以绑定          | 不能绑定       |



& 只影响绑定规则不影响 T 的类型，T 一律都会推断为非引用。

**T&&的特殊推断**：

当形参类型为 T&& 时，如果实参为左值则 T 推断为 X& 或 const X&，参数类型为 `X& p`（因为根据引用折叠，X& &&t 折叠为 X &t，参数按引用传递）；如果实参为右值则 T 推断为 X，参数类型为 `X && p`。

**折叠引用**：

*通常情况下不能定义引用的引用，通过类型别名或通过模板类型参数间接定义是可以的*

- X& & 、X& && 、X&& & 都折叠为 X&
- X&& && 折叠为 X&& 

折叠引用只能应用于间接创建的的引用的引用，如类型别名或模板参数

```cpp
template<typename T>
void f(T && a) {
    a = 4;
    // T b = 3;  //  f(i) ：non-const lvalue reference to type 'int' cannot bind to a temporary of type 'int'
    // T b = 3; // f(1) : OK
}

int main(void) {
    int i = 3;
    f(i); // 实参为左值 int， T 为 int&，按引用传递
    std::cout << i << std::endl;   // i == 4;
    f(1); // 实参为右值 int，T 为 int，按值传递
}
```

右值引用通常用于两种情况：模板转发 ([[完美转发]]) 或模板重载。

目前模板通常的形式为：

```cpp
template <typename T> void f(T&&);       // 能够绑定到非 const 右值
template <typename T> void f(const T&);  // 能够传入左值和右值
```

## std::move

**std:: move 就是左值转为右值罢了**。

```cpp
// 原理：
struct A {
    A(){
        std::cout << "A();" << std::endl;
    }
    A(const A & a) {
        std::cout << "A(const A & a);" << std::endl;
    }
    A(A && a) {
        std::cout << "A(A && a);" << std::endl;
    }
};
// 接受的参数类型为右值，右值引用 a 绑定到传入的右值实参
A&& myMove(A &&a) {
    std::cout << "A && myMove(A &&a);" << std::endl;
    // static_cast<A&&>将左值转为右值，因为 a 是具名变量
    return static_cast<A&&>(a);
    // return a; // 等价于 A && temp = a; 报错： rvalue reference to type 'A' cannot bind to lvalue of type 'A'
    // a 的类型是 A &&，同时 a 是一个具名变量，是左值
}
// 接受的参数类型为左值，a 绑定到传入的左值上
A&& myMove(A &a) {
    std::cout << "A && myMove(A &a);" << std::endl;
    return static_cast<A&&>(a); // 将左值转换成右值
}
int main(void) {
    A a3(myMove(A()));
    A a;
    A a4(myMove(a));
    A& ra;
    myMove(ra); // OK，这里并不是定义了引用的引用。ra 就是 a 的别名，实际上形参引用是绑定到对象 a 上了
}
```

[[函数返回机制]]

**函数返回值用于初始化调用点的一个临时变量，该临时变量就是函数调用结果**

返回引用，则该引用仅仅是它所引用对象的一个别名。

```cpp
// std::move
typename remove_reference<_Tp>::type&& move(_Tp&& t)
{
    return static_cast<typename remove_reference<_Tp>::type&&>(t);
}
```

因此，**`std::move` 基本原理就是利用万能绑定 `_Tp&&`，可以接受任何类型的左值和右值**。当函数实参为左值时 T 推断为 T&，函数形参类型为 T& && 折叠为 T&，函数接受一个左值实参；当函数实参为右值时 T 推断为 T ，函数接受的类型为 T&&，函数接受一个右值实参。

**然后利用 `static_cast<_Tp&&>(t)` 将左值形参 t 强转为 h 右 h 值，并返回右值引用**。

## 从左值 static_cast 到右值是允许的

不能隐式的将左值转换为右值（那不然右值引用既能绑定右值又能绑定左值了...），但是可以用 static_cast 将左值转换为右值，然后再绑定到一个右值引用上。

`static_cast<T&&>(expr)` 意味着使用表达式的值，而不是名字。（*个人理解*）

将左值强转为右值通常意味着**即将使用移动语义进行资源的所有权转移**。

**但是**，应该统一使用**std::move**使我们更容易找到可能会 move 左值的代码。（并不是说将左值强转为右值就会 move 左值，而是说一但使用 std:: move 那么紧接着一定会进行所有权转移，也就是调用接受 T&& 的移动构造函数。）

## 函数重载与模板


函数模板会从以下几个方面影响[[函数匹配规则]]：

- 候选函数将包括所有模板实参推断成功的模板实例
- 候选函数模板总是可行的，否则就不会实例化
- 和往常一样，可行参数按类型转换来排序
  函数模板的类型转换非常有限：const 转换，从数组或函数转换成相应的指针
- 如果有且只有一个最佳匹配，选择它
  如果有多个最佳匹配，不一定会直接报错：call to 'f' is ambiguous
  - 只有一个是非模板函数，选择此函数
  - 只有模板函数，有且只有一个更加特例化，则选择此函数

**正确定义一组重载的函数模板需要对类型间的关系及模板函数允许的有限的实参类型转换有深刻的理解**

一组例子
```cpp
template <typename T>
void f(const T &t) {std::cout << "1" <<std::endl;}
template <typename T>
void f(T *t) {std::cout << "1" <<std::endl;}

int main(void) {
    int i = 3;
    f(&i);       // 会调用哪个函数模板?
    return 0;
}

// 1. 实例化为
void f(const int * &);  // 需要的类型转换：const 转换(底层 const)，从 int*& -> const int*&
void f(int * ); //  精确匹配

int main(void) {
    const int ci = 3;
    f(&ci);       // 会调用哪个函数模板?
    return 0;
}
// 1. 实例化为
void f(const int * &);  // 精确匹配, T = int *
void f(const int * ); //  精确匹配, T = const int ，会调用模板二
// 并不是根据函数匹配规则，而是根据 “只有模板函数，有且只有一个更加特例化，则选择此函数”确定的
```

1. 只有模板函数，有且只有一个**更加特例化**，则选择此函数

**更加特例化**并不是指模板实例的形参对实参的匹配更精确，前提是有多个最佳匹配说明匹配程度相同。

**更加特例化是指模板函数对应的实例更少，能够匹配的类型更少**

`const T &` 是万能绑定，能够绑定所有类型的左值，包括指针类型。如果没有这条规则，`const T&` 和 `T *` 对传入的指针都是精确匹配，将无法调用指针版本（`T*`）的模板函数。

2. 只有一个是非模板函数，选择此函数

这条规则使得我们定义了通用的模板函数之后，还能定义它的特例版本。

```cpp
template <typename T>
void f(const T &t);

void f(const string &t);

void main(void) {
    f(string("abc"));   // 会调用普通函数
}
```

## 可变参数模板

类比与 C 中的 `...`

**参数包**：可变数目的参数

- 模板参数包：表示零个或多个模板参数
- 函数参数包：表示零个或多个函数参数

`class...` 或 `typename...` 表示后面的模板形参**Args 是一个模板参数包**

`类型名 ... ` 表示后面的函数形参是一个函数参数包 **rest 才是函数参数包**

```cpp
template <typename T, typename... Args>
void foo(const T &t, const Args&... rest);
```
**sizeof...** 运算符 *比 C 的*

`sizeof...` 可以返回包中的元素数量，返回值为 constexpr

### 可变参数的函数模板通常是递归的

```cpp
template <typename T>
std::ostream &print(std::ostream &os, const T &t) {
    return os << t << std::endl;
}
template <typename T, typename... Args> 
std::ostream &print(std::ostream &os, const T &t, const Args& ... rest) {
    os << t << ", ";
    return print(os, rest...);  // 包扩展
}
int main(void) {
    print(std::cout, 2, 3.2, std::string("abc"));
}

```
可变参数版本的 print 接受三个参数，但时每次只传入两个实参，一个 `ostream` 一个参数包。因此，每调用一次 print，rest 包中的第一个实参就会被绑定到 t，剩下实参形成下一个 print 调用的参数包。

最后一次 print 中, 也就是 `print(std::cout, std::string("abc"))` 时，两个模板的实例化都可以匹配（参数包中的参数可以为 0）。但是，**非可变参数模板比可变参数模板更特例化**。

**定义可变参数版本的 print 时，非可变参数的 print 的声明一定在作用域中，否则，可变参数版本会无限递归**

C++17 中有了**折叠表达式**

```cpp
template <typename... Args>
std::ostream& Println(Args&&... args) {
    // C++17 折叠展开表达式，在 C++11 中要是用递归
    ((std::cout << std::forward<Args>(args) << ' '), ...);
    return std::cout << std::endl;
}
```

一个特例化版本就是**模板更精确的定义，其中一个或多个模板参数被定义为特定的类型**

特例化一个函数或一个类都是以一个空的模板参数列表开始的

### 函数模板特例化和函数重载

**函数模板特例本质上是函数模板的一个实例** ，模板特例只会以模板实例的角色去影响函数重载。

[[#函数重载与模板]]

作用域规则：

- 特例化一个模板时，原模板的声明必须在在作用域中
- 使用模板实例时，模板特例的声明也必须作用域中（否则根本没可能调用特例）
- 普通类和函数丢失声明的情况很容易发现——编译器将不能继续编译，但如果丢失了一个模板特例，编译器还能用原模板生成一个实例。但最后程序运行时的行为会和我们设想的行为产生区别，这很难 Debug。
- 最佳实践：**模板声明、specializations 声明、及其实现都应该放在同一个头文件中。所有同名模板的声明应该放在前面，然后是这些模板的 specializations。**

#### 为 `std::hash` 定义一个自定义类的特例

一个例子：为 `std::hash` 定义一个自定义类的特例

`std::hash<key_type>`：用来给每个内置类型或一些标准库类型 (如`std::string`)对象生成 `hash` 值

一个特例化 hash 类必须定义

- 重载的调用运算符，接受元素类型对象，返回一个 size_t （hash 函数的本体）
- 两个类型成员，`result_type`和`argument_type`：调用运算符的返回值对象和形参对象
