## 主动断开

[9 | 优雅地关闭还是粗暴地关闭 ? - 知乎](https://zhuanlan.zhihu.com/p/337005775)
### close：粗暴

函数会对套接字引用计数减一，一旦发现套接字引用计数到 0，就会对套接字进行彻底释放，**并且会关闭 TCP 两个方向的数据流**。

*socket 引用计数*：套接字可以被多个进程共享，如果通过 fork 的方式产生子进程，套接字的引用计数会 +1

### shotdown：优雅

```cpp
int shutdown(int sockfd, int howto)
```
howto 是这个函数的设置选项，它的设置有三个主要选项：

- **SHUT_RD**：关闭连接的“读”这个方向，对该套接字进行读操作直接返回 EOF。从数据角度来看，套接字上接收缓冲区已有的数据将被丢弃，如果再有新的数据流到达，会对数据进行 ACK，然后悄悄地丢弃。也就是说，对端还是会接收到 ACK，在这种情况下根本不知道数据已经被丢弃了。
- **SHUT_WR**：关闭连接的“写”这个方向，这就是常被称为”半关闭“的连接。此时，不管套接字引用计数的值是多少，都会直接关闭连接的写方向。套接字上发送缓冲区已有的数据将被立即发送出去，并发送一个 FIN 报文给对端。应用程序如果对该套接字进行写操作会报错。
- **SHUT_RDWR**：相当于 SHUT_RD 和 SHUT_WR 操作各一次，关闭套接字的读和写两个方向。

### 区别

1. close 会关闭连接 (当 socket 的引用计数为 0 时），**并释放所有连接对应的资源，而 shutdown 并不会释放掉套接字和所有的资源。**
2. **close 存在引用计数的概念，并不一定导致该套接字不可用；shutdown 则不管引用计数，直接使得该套接字不可用，如果有别的进程企图使用该套接字，将会受到影响。**
3. shutdown 可以只关闭一端（比如写端），会向对端发送 FIN，表示不会再发送数据了，但此时**还可以接收数据**。

## 使用工具关闭某个 TCP 连接

**原理：获取连接 TCP 当前的序列号，伪造 RST 报文**

tcpkill：被动获取某个 TCP 连接当前的序列号，不能关闭非活跃连接

killcx：主动发送一个 SYN 报文，对端会会一个"Challenge ACK"，通过这种方式可以获取到 TCP 当前连接的序列号。能关闭非活跃连接。
