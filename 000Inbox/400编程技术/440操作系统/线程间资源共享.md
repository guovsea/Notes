## 线程私有资源

线程运行的本质其实就是函数的执行，函数的执行总会有一个源头，这个源头就是所谓的入口函数，CPU 从入口函数开始执行从而形成一个执行流，只不过我们人为的给执行流起一个名字，这个名字就叫线程。

既然线程的本质就是函数的执行，那么**函数执行都有哪些信息**呢？
- 栈指针
- 栈
- 程序计数器
- 寄存器

此外，还有操作系统管理线程的信息

因此，线程的**私有资源**
1. **线程 id**
2. 处理器现场和内核栈  [[用户空间栈和内核栈]]
3. 用户空间栈 -> **栈**
4. **errno 变量**
5. **信号屏蔽字**
6. **调度优先级** *线程是最小的执行单位*


**errno**

`errno`是一个在 C 和 C++标准库中定义的全局错误码变量，用于存储函数调用的错误信息。`errno`是线程本地存储（Thread-Local Storage，TLS），每个线程都有自己的独立的`errno`变量。这意味着不同线程的`errno`变量是独立的，它们不会相互干扰。

当一个线程在调用标准库函数或系统调用时设置了`errno`的值，这个值只会影响到调用线程自身。其他线程的`errno`值不受影响。因此，`errno`的线程本地存储性质使得它适用于多线程编程，不会导致线程间的竞争条件或数据不一致性问题。
