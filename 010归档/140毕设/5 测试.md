### 单元测试

本文采用 Google Test（简称 GTest）进行单元测试，GTest 是一个由 Google 开发的 C++测试框架，用于编写和自动化运行 C++ 测试。

当软件系统进行更改或添加新功能时，单元测试有助于确保现有功能没有被破坏。在添加或修改代码后可以运行所有 GTest 测试用例，用于捕捉由于代码更改引入的错误。通过针对代码的各个部分编写测试用例，可以确保每个功能模块按照预期工作，从而提高代码质量。

例如，在测试主机和主机通信时，ARP 协议是否正常工作，可以编写如下单元测试：

```cpp
TEST(TestArp, HostToHost)
{
	auto h1 = new Host("h1");
	auto h2 = new Host("h2");
	ConnectInterface(h1, "G/0", h2, "G/0");
	h1->SetIpAddress("192.168.1.1");
	h2->SetIpAddress("192.168.1.2");

	h1->SetSubnetMask("255.255.255.0");
	h2->SetSubnetMask("255.255.255.0");

	h1->SetGateway("192.168.1.2");
	h2->SetGateway("192.168.1.1");

	h1->FlushSetting();
	h2->FlushSetting();

	QObject::connect(h1, &Host::ArpTableChanged, [=]()
	{
	  EXPECT_EQ(h1->GetArpTable().size(), 1);
	  EXPECT_EQ(h1->GetArpTable()["192.168.1.2"], h2->GetInterface()->GetMacAddress());
	});

	h1->BroadcastArpRequest("192.168.1.2");

}
```
主机 1 (h1) 以 主机 2 (h2) 的 IP 地址为目标地址向主机 2 发送 ARP 请求，主机 1在收到 h2 返回的 ARP 响应后会修改它的 ARP 缓存。当 ARP 缓存被修改后断言 h1 的 ARP 缓存中是否有 h2 的 IP 地址和 MAC 地址映射关系，并且断言 h1 的 ARP 缓存中的 h2 的 MAC 地址是否和 h2 的实际 MAC 地址相同。当测试失败时，GTest 会将测试失败测试用例以及断言失败的实际值和预期值打印输出，方便后续的 debug。
