### 1. **Install Homebrew for x86_64**  


First, install a separate x86_64 version of Homebrew alongside your ARM version:  首先，安装一个


```bash
# Start a terminal in Rosetta 2 x86_64 mode
arch -x86_64 zsh

# Install Homebrew for x86_64 (if not already installed)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### 2. **Install `gdb` using the x86_64 Homebrew**

```bash
# From your x86_64 terminal session:
arch -x86_64 brew install gdb
```

### Why This Works:  为什么这有效：

- Apple Silicon Macs can run x86_64 binaries via **Rosetta 2**.  

- The x86_64 Homebrew installation allows you to use packages that haven’t been optimized for ARM yet.  

## 这样不会有冲突吗，两个 brew 分别会在什么地方？

在 Apple Silicon（M1/M2 等 ARM 芯片）的 Mac 上，不同架构的 Homebrew 会安装到不同位置：

### 1. **原生 ARM 版 Homebrew**（默认）

- **安装路径**：`/opt/homebrew`
- **适用场景**：为 ARM 架构（`arm64`）编译的软件。
- **使用方式**：直接运行 `brew` 命令（如 `brew install ...`）。

### 2. **x86_64 版 Homebrew**（通过 Rosetta 2）


### 如何区分两种 Homebrew？

如何区分两种 Homebrew？
查看安装路径：

# 原生 ARM 版 Homebrew


```
which brew        # 输出 /opt/homebrew/bin/brew
```

# x86_64 版 Homebrew

```
arch -x86_64 /usr/local/bin/brew --prefix  # 输出 /usr/local
```


---

### 为什么要分两个路径？

- **避免冲突**：ARM 和 x86_64 软件二进制不兼容，分开安装可确保两者共存。
    
- **性能优化**：ARM 软件能原生运行，x86_64 软件通过 Rosetta 2 转译运行。
