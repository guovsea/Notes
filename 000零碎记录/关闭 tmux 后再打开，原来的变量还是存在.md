### 一、原因分析

1. **tmux 的会话持久化**：
    
    - tmux 服务端进程在后台持续运行，即使你断开终端或关闭窗口，**会话中的进程（包括 Shell 及其环境变量）仍然保持存活**。
        
    - 当你重新执行 `tmux attach` 时，实际上是重新连接到这个**存活**的会话，而不是启动全新的 Shell 进程。之前通过 `export` 设置的变量自然仍然存在。
        
2. **Shell 进程的独立性**：
    
    - 每个 tmux 窗口/面板对应一个独立的 Shell 进程。如果在这个进程中设置了环境变量，**只要该进程不退出，变量会一直存在**。
        
    - 重新连接 tmux 只是重新附加到现有进程，不会触发 Shell 的重新初始化（如重新加载 `~/.bashrc` 或 `~/.zshrc`）。
        

---

### 二、验证方法

1. **检查 tmux 会话是否为新创建**：
    
    - 通过 `tmux ls` 查看现有会话列表，确认你附加的是新会话还是旧会话。
        
    - 如果会话已经存在（状态为 `attached` 或 `detached`），说明环境变量可能被保留。
        
2. **观察 Shell 进程的 PID**：
    
    - 在 tmux 窗口中执行 `echo $$` 记录进程 PID。
        
    - 断开并重新附加后再次执行 `echo $$`，如果 PID 相同，说明是同一个 Shell 进程。
        

---

### 三、解决方法

#### 方法 1：明确创建新会话

重新打开 tmux 时，**强制创建新会话**，而不是附加到旧会话：

```
tmux new -s mysession  # 创建名为 mysession 的新会话
```
#### 方法 2：退出时清理会话

在退出 tmux 前，**彻底关闭所有窗口**（确保 Shell 进程退出）：

```
exit
```

或使用快捷键 Ctrl+D
