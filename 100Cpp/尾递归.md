尾递归，进入下一个函数不再需要上一个函数的环境了，得出结果以后直接返回。如果函数返回后还需要进行某些操作，那么它就不是尾递归。

形式上只要最后一个`return`语句是单纯函数就可以

```c
return rec(x + 1) + a; // 非尾递归
return tailrec(x+1);  // 尾递归
```
**原理**

尾递归的递归过程中，进入下一层函数后，**上一层函数就不需要再进行任何操作**，因此，上一层函数的栈帧并不在需要保留，会被替换成下一层函数的栈帧。因此，递归过程中始终只需要保存一个栈帧。

当编译器检测到一个函数调用是尾递归的时候，它就覆盖当前的栈帧而不是在栈中去创建一个新的栈帧。


**注意**

对于 C++ 等语言来说，在函数最后`return f(x);`并不一定是尾递归——在返回之前很可能涉及到对象的析构函数，使得 f (x)`不是最后执行的那个。这可以通过 [[返回值优化 RVO]] 来解决。

下面的函数就不是尾递归，因为 return 之后还需要对 d 进行析构
```cpp
// 1累加到n
int acc(int n, int sum) {
    vector<int> vec;
    if (n == 1) {
        return sum + n;
    } else {
        return acc(n - 1, sum + n);
        // 还要对vec进行析构
    }
}
```
