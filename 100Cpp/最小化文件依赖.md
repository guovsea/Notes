## 主要方法

1. pimpl idiom

只暴露 handle，隐藏类的实现

2. Interface 

创建只具有 pure virtual 的功能函数和 virtual destructor

## 注意事项

 1. 提供接口时，应将接口和数据结构的定义分离，客户代码按需 include 数据结构定义
 
    提供接口时，应将接口和数据结构的定义分离，接口中使用到的类只需要**前置声明**。如果客户代码需要访问数据结构的定义，由客户端主动 include 数据结构的定义。
    
    ```cpp
    // interface.h
    class Date;
    class Person;
    Date get_birthday(Person persion);
    Date today();
    size_t get_age(Date date);
    
    ```

    ```cpp
    // date_def.h
    class Date{
    
    };
 
    ```
    
     客户端可以按需 include 数据定义。如果客户代码只使用 `get_age(today())` 时，只需要 include `date_def.h` 而不需要 include 其他数据定义。
    
    因此，客户代码应有选择 include 哪些类定义文件的自由。
