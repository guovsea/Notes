在 C++ 中，虚函数（virtual function）是一个类的成员函数，可以在派生类中重写（override）。**在使用虚函数时，不应该使用默认参数**。这是因为：

 **默认参数会被静态绑定，虚函数在运行时进行动态绑定**。如果虚函数使用了默认参数，那么**编译器在编译期间会根据对象的静态类型来确定默认参数的值**，这个值不会根据对象的实际类型而改变。

> **一句话总结**：虚函数是运行期多态，默认参数是编译期替换，override 不会覆盖默认参数，vtable 里只有函数地址没有参数信息。

## 代码示例

```cpp
#include <iostream>

class Base {
public:
    virtual void foo(int x , int y = 5) {
        std::cout << "Base::foo(" << x << "," << y<< ")" << std::endl;
    }
};

class Derived : public Base {
public:
    virtual void foo(int x, int y = 3) override {
        std::cout << "Derived::foo(" << x << "," << y<< ")" << std::endl;
    }
};

int main() {
    Base* b = new Derived();
    b->foo(5); // Output: Derived::foo(5,5)
    b->foo(5, 2); // Output: Derived::foo(5,2)
    delete b;
    return 0;
}

```

可以看到：**被调用的函数是定义在对象的实际类型中的虚函数**这一动态特性还是成立的；但形参却被赋值为**对象的静态类型中声明的虚函数的默认值**

> 如果不依赖默认参数则没有任何影响（意味着会产生难以发现的 bug）

**函数形参默认的绑定发生在编译期间，而不是运行期间**

## 原理

函数调用时，参数的数量、类型、使用哪些寄存器或栈位置，全部由 ABI 和调用点在编译期决定。若使用了缺省参数，编译器会在调用点直接将缺省值替换为立即数或变量，放入相应的寄存器或栈中。这一过程与函数是否为虚函数无关。

对于虚函数调用，编译器在编译期确定的是虚函数在虚函数表中的槽位（即相对于 vtable 的固定偏移），而不是具体调用哪一个函数实现。运行时通过对象的地址取得其虚函数表指针，再根据该固定偏移取出函数地址并调用，从而实现多态。

## 虚函数&函数默认实参的汇编代码


```cpp
class Base {
public:
    virtual void foo(int x , int y = 5) {
    }
    virtual void bar() {}
};

class Derived : public Base {
public:
    virtual void foo(int x, int y = 3) override {
    }
    virtual void bar() override {}
};
void f(Base* b) {
    b->foo(2);
    b->bar();
}

```

平台假设：**x86-64 System V ABI（Linux / GCC / Clang）**，`-O0`。

## 一、先对照 C++ 代码

```cpp
void f(Base* b) {
    b->foo(2);
    b->bar();
}
```

关键点先记住：

1. `b` 的**静态类型是 `Base*`**

2. `foo` 是虚函数，但

3. **默认参数在编译期决定**

4. vtable 布局（逻辑上）：


```
vtable:
  [0] foo
  [1] bar
```

---

## 二、函数入口：建立栈帧

```asm
f(Base*):
 push   rbp
 mov    rbp,rsp
 sub    rsp,0x10
```

### 含义

1. `push rbp`
    保存调用者的栈基址

2. `mov rbp, rsp`
    建立当前函数的栈帧

3. `sub rsp, 0x10`
    分配 16 字节栈空间（局部变量 + 对齐）


---

## 三、保存参数 `b`

```asm
mov    QWORD PTR [rbp-0x8], rdi
```

### 含义

- **System V ABI 规定**

    - 第 1 个参数 -> `rdi`

- 这里的参数是 `Base* b`


> 把 `b` 保存到栈上：

```
[rbp-0x8] = b
```

---

## 四、准备调用 `b->foo(2)`

### 1. 取出 `b`

```asm
mov    rdi, QWORD PTR [rbp-0x8]
```

- `rdi = b`

- **虚函数调用时**

    - `this` 指针必须放在 `rdi`


---

### 2. 取 vtable 指针

```asm
mov    rax, QWORD PTR [rdi]
```

内存结构：

```
b ---> [ vptr ] ---> vtable
```

所以：

- `[rdi]` = vptr

- `rax = vtable 地址`


---

### 3. 准备参数

```asm
mov    esi, 0x2
mov    edx, 0x5
```

根据 ABI：

|参数|寄存器|值|
|---|---|---|
|this|rdi|b|
|x|esi|2|
|y|edx|5|

> **关键点**

- `y = 5`

- **来自 `Base::foo` 的默认参数**

- 和 `Derived::foo(int, int=3)` 完全无关


---

### 4. 虚函数调用

```asm
call   QWORD PTR [rax]
```

等价于：

```cpp
call vtable[0]; // foo
```

> **运行期决定调用的是：**

- `Derived::foo`（如果 b 实际指向 Derived）


> **但参数 y 已经是 5 了**

---

## 五、准备调用 `b->bar()`

### 1. 再次取 `b`

```asm
mov    rdi, QWORD PTR [rbp-0x8]
```

- `this = b`


---

### 2. 再次取 vtable

```asm
mov    rax, QWORD PTR [rdi]
```

- `rax = vtable`


---

### 3. 调用第二个虚函数

```asm
call   QWORD PTR [rax+0x8]
```

解释：

- 每个 vtable 表项 8 字节（64 位地址）

- `+0x8` -> 第 2 个虚函数


```
vtable[1] = bar
```

> 调用：

```cpp
Derived::bar(); // 运行期多态
```

---

## 六、函数结束：销毁栈帧

```asm
add    rsp,0x10
pop    rbp
ret
```

- 回收局部变量

- 恢复调用者栈帧

- 返回


---

## 七、`nop`

```asm
nop
```

- 无操作

- 对齐 / 调试用

- 对语义无影响


---

## 八、`main` 为什么是空的？

```asm
main:
 push   rbp
 mov    rbp,rsp
 mov    DWORD PTR [rbp-0x4],0x0
 xor    eax,eax
 pop    rbp
 ret
```

### 原因

- `main` 没有调用 `f`

- 没有可观测副作用

- 编译器在 `-O0` 下仍保留最小结构


---

## 九、把"汇编行为"和"C++ 语义"一一对应

### 1. 虚函数体现在哪里？

```asm
mov rax, [rdi]      ; vptr
call [rax]          ; foo
call [rax+0x8]      ; bar
```

> **运行期通过 vtable 决定函数地址**

---

### 2. 默认参数体现在哪里？

```asm
mov edx, 0x5
```

> **编译期直接写死**
> **和 vtable、override 完全无关**

---

## 十、总结

> **虚函数：运行期多态（call [vtable+offset]）**
> **默认参数：编译期替换（mov edx, 5）**
> **override 不会覆盖默认参数**
> **vtable 里只有函数地址，没有参数信息**
